## طبيعة المادة

| النوع | الاستخدام | أمثلة |
| --- | --- | --- |
| نظري | مفاهيم التزامن والاتساق والتصميم الموزع | `Consistency Models`، `CAP Theorem`، `Consensus` |

## القواعد الإلزامية

- لا تتجاهل أي سطر أو معلومة وردت في المحاضرة
- أكمل الناقص مع وسم **(شرح زيادة للفهم)** أو **(غير مشروحة في المحاضرة)**
- ابدأ من المبتدئ، لا تنتقل لنقطة قبل إتمام شرح السابقة
- اشرح **لماذا** وراء كل فكرة، لا التعريف فقط
- تشبيه يومي + مثال عملي بعد كل نقطة
- اتبع تسلسل المحاضرة نفسها

## 🎯 أنواع أسئلة إضافية إلزامية للأتمتة (الفحص بالكامل آلي)

> هذه الأنواع تُدمَج **ضمن** الأجزاء أعلاه (لا تُنشئ لها أقساماً ## منفصلة) — التزم بها لضمان كفاية وتنوّع الأسئلة القابلة للتصحيح الآلي.

### مراجعة سريعة (سؤال / جواب) — ضمن qa_cards / qa_inline / theory (≥12، easy)
**Q{N}:** سؤال مراجعة مباشر (تعريف، فرق، متى نستخدم). **A:** إجابة مختصرة قابلة للمقارنة الآلية.
مثال: «ما الفرق بين النموذج `Client-Server` والنموذج `Peer-to-Peer`؟»

### تحدي مركّب (سيناريو + أسئلة متعددة) — ضمن mcq / theory (≥3، hard)
### السيناريو {N}: {عنوان}
> [وصف الحالة: جدول تكرارات، مخطط زمني، أو مسألة رياضية كاملة — 5–10 أسطر]
ثم 3–5 أسئلة MCQ **كلها مبنية على نفس السيناريو**.
المصادر: `code_trace`، `numerical_case`، `algorithm_state`، `logic_entailment`، `system_design`.
مثال: «جدول تكرارات خوارزمية `Lamport` — أي قيمة في الخلية X؟»

### تطبيق عملي (أكمل / صحّح / نفّذ) — ضمن exercise / analysis_exercise (≥5، medium)
كل تمرين: **المعطيات** + **المطلوب** (مرقّم) + **نموذج الحل الكامل**.
الأنواع: `fill_gaps`، `code_fix`، `table_fill`، `numerical_solve`، `diagram_completion`.

### تتبع خطوة بخطوة (جدول ناقص) — ضمن trace_exercise / mcq (≥3، medium)
**المدخل:** {حالة ابتدائية} → **أكمل الجدول** → **نموذج الحل**.
الأهداف: `algorithm_iterations`، `variable_state`، `search_tree_expansion`، `memory_page_replacement`.

### مقارنة وتحليل — ضمن mcq / theory / analysis_exercise (≥4، medium)
سؤال مقارنة بخيارات أو جدول: | المعيار | الخيار A | الخيار B |

### اكتشف الخطأ (كود / منطق / حساب) — ضمن exercise (≥4، hard)
**الكود/الحساب (يحتوي خطأ):** … **اكتشف الخطأ:** … **التصحيح:** … + شرح الحل (3 نقاط).
أنواع الأخطاء: `logic`، `misconception`، `off_by_one`، `wrong_formula`، `race_condition`، `infinite_loop`.

### نظري منظم — ضمن theory / mcq (≥8، medium)
### السؤال {N}: {السؤال}
**نموذج الإجابة:** 1. التعريف 2. المكونات/الشروط 3. مثال 4. متى نستخدم.

## قواعد الكتل داخل الشرح

**📊 المخطط** — 3 أقسام بالترتيب: ما هذا المخطط؟ + جدول العُقد + جدول الروابط + بلوك `diagram`.
types: `flowchart`، `bpmn`، `decision-tree`، `dfd`. انظر SCHEMA.md v1.0 §Diagram.

**⚙️ الخطوات / الخوارزمية** — أسطر داخل fence ` ```algorithm ` بصيغة `1 | الخطوة | الأداة | ماذا يحدث` (سطر لكل خطوة). لا ترسم مربعات ولا أسهم — البارسر يحوّلها. انظر SCHEMA.md v1.0 §Algorithm.

**pseudo-code** — داخل code fence بـ `language_tag=text`.

**الفهم الخاطئ ❌** / **الفهم الصحيح ✅** — سطر واحد لكل منهما.

#### مهم للامتحان ⚠️ / #### نقطة مهمة ⚠️ / #### ملاحظة: / #### الدرس المستفاد:
← `blockquote` يتبع كل منها مباشرةً.

**🤔 تفعيل الفهم** — استخدمه **≥3 مرات** في كل محاضرة.

كود ناقص — ضع `_______` أو `// (N)` مكان السطر الناقص.

تصحيح كود — قدّم الكود الخاطئ أولاً ثم المصحّح.

**Q{N}:** سؤال / **A:** إجابة.

**💡 التشبيه** — بعد كل مفهوم مجرّد: جملة واحدة من الحياة اليومية + «وجه الشبه: X = Y». استخدمه بكثرة. انظر SCHEMA.md v1.0 §Analogy.

**⚖️ المقايضة** — جدول: المزايا | العيوب | متى تختاره. للحالات التي لا يوجد فيها خطأ وصواب. انظر SCHEMA.md v1.0 §Trade-off.

**🔄 قبل / بعد** — حالة قبل العملية + بعدها + جملة «ماذا تغيّر؟». انظر SCHEMA.md v1.0 §Before-After.

**🔍 تتبع التنفيذ** (inline) — المدخل + جدول الخطوات + النتيجة. انظر SCHEMA.md v1.0 §Trace.

**📐 المعادلة** — LaTeX داخل `$$` أو fence `math`. يتبعها **الشرح:** بمعنى كل رمز (`Latency`، `Bandwidth`، `Jitter`، `Clock Skew`، `Clock Drift`، `Lamport Timestamp`، `Vector Clock`). انظر SCHEMA.md v1.0 §Equation.
