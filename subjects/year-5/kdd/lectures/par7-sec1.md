### 1. مقدمة عن طرق التصنيف الأساسية

#### النص الأصلي يقول:
> "Basic Techniques For Data Classification: Bayes Classification Methods, Nearest-neighbor, Support Vector Machines, Rule-Based Classification"

#### الشرح المبسّط:
هذه المحاضرة جزء من سلسلة أكبر عن طرق `Classification` (التصنيف). سبق أن رأينا `Decision Tree` (شجرة القرار)، والآن ندخل إلى عائلة جديدة كاملة من الطرق: الطرق **الاحتمالية (Probabilistic)** المبنية على `Bayes' Theorem` (نظرية بايز).

**لماذا؟** لأن شجرة القرار تتخذ قراراً حاسماً (نعم/لا) بناءً على تقسيمات متتالية، بينما طريقة بايز تُعطي **احتمالاً** لكل صنف ثم تختار الأعلى احتمالاً — هذا مفيد جداً عندما تريد معرفة "ما مدى ثقتنا" بالقرار، وليس فقط القرار نفسه.

#### 💡 التشبيه:
> تخيل طبيباً يشخّص مريضاً: شجرة القرار تسأل أسئلة متتالية (هل الحرارة مرتفعة؟ → هل هناك سعال؟) لتصل لتشخيص واحد قاطع. أما طريقة بايز فهي أشبه بطبيب يقول: "بناءً على الأعراض، احتمال أن يكون المرض A هو 70%، والمرض B هو 30%" ثم يختار الأعلى.
> **وجه الشبه:** الأعراض = السمات (Attributes)، المرض المختار = الصنف (Class) صاحب أعلى احتمال.

### 2. Bayes Classification Methods — الإطار العام

#### النص الأصلي يقول:
> "A probabilistic framework for solving classification problems. predict class membership probabilities for a given tuple → The tuple class is the most likely class (Maximum Likelihood classification). A popular method for e.g.,: text classification, sentiment analysis. based on Bayes' theorem."

#### الشرح المبسّط:
`Bayes Classification` هو **إطار احتمالي (probabilistic framework)** لحل مشاكل التصنيف. الفكرة الأساسية: لكل سجل بيانات (Tuple) نحسب **احتمال انتمائه لكل صنف ممكن**، ثم نختار الصنف صاحب **أعلى احتمال** — وهذا ما يسمى `Maximum Likelihood Classification` (تصنيف الاحتمال الأعظم).

**لماذا هذه الطريقة مهمة؟** لأنها بسيطة رياضياً، سريعة الحساب، وتعمل بشكل ممتاز في `text classification` (تصنيف النصوص) و `sentiment analysis` (تحليل المشاعر) — مثل تصنيف رسالة بريد إلكتروني كـ Spam أو ليست Spam بناءً على الكلمات الموجودة فيها.

#### 💡 التشبيه:
> مثل شخص يقرر ماذا يرتدي غداً بناءً على احتمال هطول المطر: إذا كان احتمال المطر 80%، فهو "يصنّف" الغد على أنه "يوم ماطر" ويأخذ مظلة — يختار الاحتمال الأعلى.
> **وجه الشبه:** احتمال هطول المطر = `P(class|X)`، القرار (أخذ مظلة) = التصنيف النهائي.

#### الأنواع الرئيسية:
1. **`Naïve Bayes classifiers`**: تفترض `class-conditional independence` (استقلالية شرطية بين السمات) لتبسيط الحسابات — ولهذا تُسمّى "ساذجة" (Naïve).
2. **`Bayesian Belief Networks`**: نماذج بيانية (Graphical models) تنمذج `dependencies` (الترابطات) بين السمات، أي لا تفترض الاستقلالية.

#### 🤔 تفعيل الفهم (اسأل نفسك):
> **سؤال:** لماذا تسمى Naïve Bayes بهذا الاسم رغم أنها من أكثر الطرق فعالية؟
> **لماذا هذا مهم؟** لأن فهم "لماذا ساذجة" يوضح متى تفشل الطريقة (عندما تكون السمات مترابطة فعلياً وليست مستقلة).

### 3. نظرية بايز (Bayes' Theorem): الأساسيات

#### النص الأصلي يقول:
> "P(H|X) = P(X|H)P(H) / P(X)"

#### الشرح المبسّط:
هذه هي المعادلة الأم لكل الفصل. دعنا نفكك كل رمز:

- **`X`**: سجل بيانات (Tuple) غير معروف صنفه.
- **`H`**: فرضية (Hypothesis) أن `X` ينتمي لصنف معيّن `C`.
- **`P(H|X)`**: الاحتمال البعدي (`posterior probability`) — احتمال صحة الفرضية `H` بعد أن رأينا البيانات `X`. هذا هو ما **نريد حسابه**.
- **`P(H)`**: الاحتمال القبلي (`prior probability`) لـ `H` — احتمال صحة الفرضية بغض النظر عن `X`.
- **`P(X|H)`**: احتمال ملاحظة العينة `X` بافتراض أن `H` صحيحة.
- **`P(X)`**: الاحتمال القبلي لـ `X` — احتمال ملاحظة هذه البيانات بشكل عام.

**لماذا نحتاج هذه النظرية؟** لأننا نريد حساب `P(H|X)` مباشرة، لكن هذا صعب. النظرية تتيح لنا حسابه من كميات أسهل في التقدير من بيانات التدريب: `P(X|H)` و `P(H)`.

#### 💡 التشبيه:
> تخيل طبيباً يريد معرفة "احتمال إصابتك بمرض معين بعد رؤية أعراضك" (`P(H|X)`) — وهذا صعب مباشرة. لكن يسهل عليه معرفة أمرين: "ما نسبة انتشار هذا المرض بشكل عام" (`P(H)`) و"ما احتمال ظهور هذه الأعراض لو كنت مصاباً بالفعل" (`P(X|H)`). نظرية بايز تجمع هذين الأمرين ليحسب ما يريد.
> **وجه الشبه:** الأعراض = X، المرض = H.

#### 📐 المعادلة: Bayes' Theorem
$$
P(H|X) = \frac{P(X|H)\,P(H)}{P(X)}
$$

**الشرح:**
> `P(H|X)` = الاحتمال البعدي المطلوب | `P(X|H)` = احتمال المشاهدة بشرط الفرضية | `P(H)` = الاحتمال القبلي للفرضية | `P(X)` = الاحتمال القبلي للمشاهدة (ثابت لجميع الفرضيات المقارنة بينها).

#### مثال (1): تفسير الرموز عملياً
**السيناريو:** بيانات زبائن موصوفة بسمتي `age` و `income`، و`X` هو زبون عمره 35 سنة ودخله 40,000$. الفرضية `H`: الزبون سيشتري حاسوباً (`buys_computer`).

| الرمز | المعنى في هذا المثال |
| --- | --- |
| `P(H\|X)` | احتمال أن يشتري الزبون X (35 سنة، دخل 40K) حاسوباً |
| `P(H)` | احتمال أن يشتري **أي** زبون حاسوباً بغض النظر عن العمر والدخل |
| `P(X\|H)` | احتمال أن يكون الزبون 35 سنة ودخله 40K **بشرط أنه اشترى** حاسوباً |
| `P(X)` | احتمال أن يكون أي زبون من مجموعتنا عمره 35 ودخله 40K |

#### مثال (2): حساب فعلي (Meningitis / Stiff Neck)
**المعطيات:**
- `P(S|M) = 0.5` (احتمال تصلب الرقبة بشرط الإصابة بالتهاب السحايا)
- `P(M) = 1/50000` (احتمال الإصابة بالتهاب السحايا)
- `P(S) = 1/20` (احتمال تصلب الرقبة بشكل عام)

**المطلوب:** إذا كان المريض يعاني من تصلب الرقبة، ما احتمال إصابته بالتهاب السحايا؟ أي احسب `P(M|S)`.

```math
P(M|S) = P(S|M) * P(M) / P(S) = (0.5 * 1/50000) / (1/20) = 0.0002
```

**الشرح:** رغم أن تصلب الرقبة عرَض شائع لالتهاب السحايا (50%)، فإن احتمال الإصابة الفعلية يبقى ضئيلاً جداً (0.02%) لأن التهاب السحايا مرض نادر أصلاً (`P(M)` صغير جداً). هذا يوضح أهمية `prior probability` في النتيجة النهائية.

#### مهم للامتحان ⚠️:
> لا تخلط بين `P(H|X)` و `P(X|H)` — هما مختلفان تماماً! هذا الخطأ الشائع يسمى "مغالطة النائب" (Prosecutor's fallacy) في الإحصاء.

### 4. المصنّف البايزي الساذج (Naïve Bayesian Classifier)

#### النص الأصلي يقول:
> "Let D be a training set... each tuple X is represented by n-dimensional attribute vector, X=(A1,A2,…,An). there are m classes, C={c1,c2,…,cm}. Given a tuple, X, the Bayesian classifier will predict that X belongs to the class having the highest posterior probability conditioned on X."

#### الشرح المبسّط:
لدينا مجموعة تدريب `D`، كل سجل `X` هو متجه من `n` سمة: `X = (A1, A2, ..., An)`. ولدينا `m` صنف ممكن: `C = {c1, c2, ..., cm}`.

**الهدف:** إيجاد الصنف `c` الذي يجعل `P(c|X)` أكبر ما يمكن — أي:

```math
c = argmax_{c in C} P(c|X)
```

بتطبيق نظرية بايز:

```math
c = argmax_{c in C} [P(X|c) * P(c) / P(X)]
```

**لماذا يمكننا حذف `P(X)` من المعادلة؟** لأن `P(X)` هو نفسه لكل الأصناف التي نقارن بينها (هو ثابت لا يتغير باختلاف `c`)، فلا يؤثر على تحديد أيّ صنف هو الأكبر. لذلك تُختصر المعادلة إلى:

```math
c = argmax_{c in C} P(X|c) * P(c)
```

#### 💡 التشبيه:
> عندما تقارن أسعار منتجين لتشتري الأرخص، وكلاهما عليه ضريبة ثابتة نفسها، يمكنك تجاهل الضريبة تماماً في المقارنة لأنها لن تغيّر أيهما أرخص.
> **وجه الشبه:** الضريبة الثابتة = `P(X)`، سعر المنتج = `P(X|c)P(c)`.

---

### 4.1 الافتراض الساذج (Naïve Assumption) وتبسيط الحساب

#### النص الأصلي يقول:
> "To reduce computation in evaluating P(X|c), A simplified assumption is made: 'The attributes' values are conditionally independent of one another, given the class label of the tuple'"

#### الشرح المبسّط:
حساب `P(X|c)` مباشرة (أي احتمال كل التركيبة من السمات معاً) صعب جداً إذا كان عدد السمات كبيراً، لأننا نحتاج بيانات ضخمة لتغطية كل التركيبات الممكنة. لذلك، `Naïve Bayes` تفترض أن **السمات مستقلة شرطياً عن بعضها البعض بشرط معرفة الصنف** — أي معرفة قيمة سمة واحدة لا تعطينا معلومة عن قيمة سمة أخرى، طالما نعرف الصنف.

**لماذا نسمّيها "ساذجة"؟** لأن هذا الافتراض نادراً ما يكون صحيحاً 100% في الواقع (مثلاً العمر والدخل غالباً مترابطان)، لكن تجاهل هذا الترابط يبسّط الحساب بشكل كبير وغالباً يعطي نتائج جيدة عملياً رغم "سذاجة" الافتراض.

بفضل هذا الافتراض:

```math
P(X|c) = P(A1|c) * P(A2|c) * ... * P(An|c) = product_{i=1}^{n} P(Ai|c)
```

وبالتالي التصنيف النهائي:

```math
c = argmax_{c in C} P(c) * product_i P(Ai|c)
```

حيث `P(c) = |c| / |D|` — أي عدد سجلات التدريب المنتمية للصنف `c` مقسوماً على العدد الكلي للسجلات في `D`.

#### ⚖️ المقايضة: افتراض الاستقلالية

| | **افتراض الاستقلالية (Naïve)** | **عدم الافتراض (Bayesian Networks)** |
| --- | --- | --- |
| المزايا | حساب سريع جداً، يحتاج بيانات تدريب أقل | يمثل الواقع بدقة أكبر عند وجود ترابط |
| العيوب | قد يفقد دقة إذا كانت السمات مترابطة فعلياً | حساب أعقد بكثير، يحتاج بيانات أكبر |
| متى تختاره | بيانات كبيرة الأبعاد (نصوص) وسرعة مطلوبة | عندما نعرف/نشك بوجود ترابط قوي بين السمات |

#### الفهم الخاطئ الشائع ❌: "Naïve Bayes تفترض أن كل السمات لها نفس الأهمية أو نفس التوزيع."
#### الفهم الصحيح ✅: الافتراض الوحيد هو **الاستقلالية الشرطية** بين السمات بشرط الصنف؛ لكل سمة توزيعها الخاص واحتمالها الخاص المحسوب على حدة.

### 4.2 حساب P(Ai|c) للسمات الفئوية (Categorical)

#### النص الأصلي يقول:
> "If Ak is categorical, then P(Ai|c) = |Ai,c| / |Dc|"

#### الشرح المبسّط:
إذا كانت السمة `Ak` **فئوية** (categorical) — مثل `Marital Status` بقيم Single/Married/Divorced — فإن حساب `P(Ai|c)` بسيط: هو **عدد السجلات التي لها قيمة السمة `Ai` وتنتمي للصنف `c`** مقسوماً على **العدد الكلي لسجلات الصنف `c`**.

```math
P(Ai|c) = |Ai,c| / |Dc|
```

حيث `|Ai,c|` = عدد السجلات التي قيمة السمة فيها = Ai وتنتمي للصنف c، و`|Dc|` = عدد كل سجلات الصنف c.

**مثال من جدول Evade:** بين سجلات الصنف `No` (7 سجلات)، عدد من حالتهم `Married` هو 4 → `P(Status=Married|No) = 4/7`. وبما أنه لا يوجد أي سجل صنفه `Yes` وحالة استرداد `Refund=Yes` → `P(Refund=Yes|Yes) = 0`.

#### 🛠️ استكشاف الأخطاء
| الخطأ | السبب | الحل |
| --- | --- | --- |
| احتمال = 0 لسمة واحدة يجعل الناتج كله صفراً | الضرب في صفر يُصفّر كل حاصل الضرب `P(X\|c)` | استخدام **`Laplace Smoothing`** (تنعيم لابلاس) لتفادي الأصفار — إضافة 1 للبسط وعدد الفئات للمقام |
| نسيان القسمة على `\|Dc\|` الصحيح لكل صنف | الخلط بين `\|D\|` الكلي و`\|Dc\|` الخاص بالصنف | التأكد من فلترة السجلات حسب الصنف أولاً |

### 4.3 حساب P(Ai|c) للسمات المستمرة (Continuous)

#### النص الأصلي يقول:
> "If Ak is continuous-valued: Discretization... Probability density estimation: Assume the attribute follows a known distribution... e.g. assume Gaussian (normal) distribution, so that P(xk|c) = g(xk, μc, σc)"

#### الشرح المبسّط:
إذا كانت السمة **مستمرة** (continuous) — مثل `Income` أو `Taxable Income` — لا يمكن ببساطة عدّ التكرارات لأن كل قيمة تقريباً فريدة. هناك طريقتان:

1. **`Discretization`**: تحويل السمة المستمرة إلى فئات (Bins) مثل [0-50K], [50-100K]... ثم التعامل معها كسمة فئوية.
2. **`Probability Density Estimation`**: افتراض أن السمة تتبع توزيعاً احتمالياً معروفاً (غالباً `Gaussian/Normal Distribution`)، ثم تقدير `μ` (المتوسط) و`σ` (الانحراف المعياري) من بيانات التدريب لهذا الصنف، واستخدام دالة الكثافة الاحتمالية مباشرة.

#### 📐 المعادلة: التوزيع الطبيعي (Gaussian)
$$
g(x, \mu, \sigma) = \frac{1}{\sqrt{2\pi}\,\sigma} \, e^{-\frac{(x-\mu)^2}{2\sigma^2}}
$$

**الشرح:**
> `x` = القيمة المراد حساب احتمالها | `μ` = متوسط قيم هذه السمة لسجلات الصنف c | `σ` = الانحراف المعياري لقيم هذه السمة لسجلات الصنف c | الناتج = كثافة احتمالية (وليست احتمالاً مباشراً بين 0 و1، لكنها تُستخدم بنفس الطريقة في المقارنة).

#### مثال رقمي:
لسمة `Income` وصنف `No`: `متوسط العينة μ = 110`، `تباين العينة σ² = 2975`.

```math
P(Income=120|No) = 1/sqrt(2π*2975) * e^(-(120-110)^2 / (2*2975)) = 0.0072
```

#### 💡 التشبيه:
> بدل أن نعدّ كم شخصاً دخله بالضبط 120,000$ (قد لا يوجد أحد بهذا الرقم بالضبط)، نرسم "منحنى جرس" (Bell Curve) يمثل توزيع الدخل، ثم ننظر لارتفاع المنحنى عند نقطة 120,000 — هذا الارتفاع هو تقدير الاحتمال.
> **وجه الشبه:** منحنى الجرس = التوزيع الطبيعي Gaussian، الارتفاع عند نقطة معينة = P(Ai|c).

#### ⚙️ الخطوات / الخوارزمية: Naïve Bayes Classification
> **الهدف:** تصنيف سجل جديد X إلى الصنف الأكثر احتمالاً بناءً على بيانات التدريب.
```algorithm
1 | حساب الاحتمال القبلي لكل صنف | العدّ على D | P(c) = |c|/|D| لكل صنف c
2 | تحديد نوع كل سمة | فحص البيانات | تصنيف السمة كفئوية أو مستمرة
3 | حساب P(Ai|c) للسمات الفئوية | العدّ الشرطي | P(Ai|c) = |Ai,c|/|Dc|
4 | حساب P(Ai|c) للسمات المستمرة | دالة Gaussian | تقدير μ,σ لكل صنف ثم حساب g(x,μ,σ)
5 | ضرب كل الاحتمالات | حاصل الضرب | P(X|c) = product P(Ai|c)
6 | ضرب النتيجة في P(c) | الضرب النهائي | Score(c) = P(c) * P(X|c)
7 | اختيار الصنف الأعلى Score | argmax | X ينتمي للصنف صاحب أعلى Score(c)
```
#### نقاط التنفيذ:
- إذا كان `P(Ai|c) = 0` لأي سمة، يُصفّر الناتج بالكامل ما لم نستخدم `Laplace Smoothing`.
- يجب حساب الاحتمالات **لكل صنف على حدة** قبل المقارنة.

### 5. مثال (1) الكامل: buys_computer

#### النص الأصلي يقول:
> "Let The class label attribute, buys_computer where C1 = yes and C2 = no. we wish to classify The tuple X where: age=youth, income=medium, student=yes, credit_rating=fair"

#### الشرح المبسّط والحل الكامل:

**الخطوة 1: الاحتمالات القبلية**
```math
P(buys_computer=yes) = 9/14 = 0.643
P(buys_computer=no)  = 5/14 = 0.357
```

**الخطوة 2: الاحتمالات الشرطية لكل سمة**

| السمة = القيمة | بشرط yes | بشرط no |
| --- | --- | --- |
| age=youth | 2/9 = 0.222 | 3/5 = 0.600 |
| income=medium | 4/9 = 0.444 | 2/5 = 0.400 |
| student=yes | 6/9 = 0.667 | 1/5 = 0.200 |
| credit_rating=fair | 6/9 = 0.667 | 2/5 = 0.400 |

**الخطوة 3: حساب P(X|c) لكل صنف**
```math
P(X|yes) = 0.222 * 0.444 * 0.667 * 0.667 = 0.044
P(X|no)  = 0.600 * 0.400 * 0.200 * 0.400 = 0.019
```

**الخطوة 4: الضرب في الاحتمال القبلي والمقارنة**
```math
P(X|yes) * P(yes) = 0.044 * 0.643 = 0.028
P(X|no)  * P(no)  = 0.019 * 0.357 = 0.007
```

**النتيجة:** بما أن `0.028 > 0.007`، فإن المصنّف يتنبأ بأن **`buys_computer = yes`** لهذا الزبون.

#### 🔍 تتبع التنفيذ: buys_computer

**المدخل:** age=youth, income=medium, student=yes, credit_rating=fair

| الخطوة | العملية | الحالة |
| --- | --- | --- |
| 1 | حساب P(yes) وP(no) | 0.643 و 0.357 |
| 2 | حساب P(Ai\|yes) لكل سمة | 0.222, 0.444, 0.667, 0.667 |
| 3 | حساب P(Ai\|no) لكل سمة | 0.600, 0.400, 0.200, 0.400 |
| 4 | ضرب P(X\|yes)*P(yes) | 0.028 |
| 5 | ضرب P(X\|no)*P(no) | 0.007 |

**النتيجة:** `buys_computer = yes` (لأن 0.028 > 0.007)

### 6. مثال (2): PlayTennis (سمات جوية)

#### النص الأصلي يقول:
> "Test instance X: Outlook=Sunny, Temperature=Cool, Humidity=High, Wind=Strong, Play=?"

#### الشرح المبسّط:
نفس المبدأ تماماً يُطبّق هنا على بيانات الطقس (14 سجل تدريب) للتنبؤ هل سيُلعب التنس أم لا:

```math
P(O="sunny"|yes) = 2/9    P(T="cool"|yes) = 3/9
P(H="high"|yes)  = 3/9    P(W="strong"|yes) = 3/9
P(yes) = 9/14
```

نضرب جميعها معاً للحصول على `P(X|yes)*P(yes)`، ثم نكرر العملية لـ `no` بنفس الطريقة (باستخدام إحصائيات الصنف `no` بدلاً من `yes`)، ثم نقارن الناتجين لتحديد الصنف الأعلى — تماماً كما في المثال السابق.

#### مهم للامتحان ⚠️:
> لاحظ أن `P(X)` في المقام (الظاهرة في الصور الأصلية للمحاضرة) لا داعي لحسابها فعلياً؛ فهي نفس القيمة للمقارنة بين yes وno، لذا يمكن الاكتفاء بمقارنة `P(X|c)*P(c)` فقط.

### 7. مثال (3): تصنيف الثدييات (Mammals)

#### النص الأصلي يقول:
> "P(X|M) = 6/7 × 6/7 × 2/7 × 2/7 = 0.06 ... P(X|N) = 1/13 × 10/13 × 3/13 × 4/13 = 0.0042 ... P(X|M)P(M) = 0.06×7/20=0.021 ... P(X|N)P(N)=0.004×13/20=0.0027 → P(A|M)P(M) > P(A|N)P(N) → Mammals"

#### الشرح المبسّط:
سجل الاختبار: `Give Birth=yes, Can Fly=no, Live in Water=yes, Have Legs=no`. يوجد صنفان: `Mammals (M)` و`non-mammals (N)`، من أصل 20 سجل تدريب: 7 ثدييات و13 غير ثدييات.

**الحساب:**
```math
P(X|M) = 6/7 * 6/7 * 2/7 * 2/7 = 0.06
P(X|N) = 1/13 * 10/13 * 3/13 * 4/13 = 0.0042
P(M) = 7/20 = 0.35     P(N) = 13/20 = 0.65
P(X|M)*P(M) = 0.06 * 0.35 = 0.021
P(X|N)*P(N) = 0.0042 * 0.65 = 0.0027
```

**النتيجة:** بما أن `0.021 > 0.0027` → التصنيف: **`Mammals`**.

#### 🤔 تفعيل الفهم (اسأل نفسك):
> **سؤال:** لماذا كان لدولفين (Give Birth=yes, Can Fly=no, Live in Water=yes, Have Legs=no) تصنيف صحيح كـ Mammal رغم أنه يعيش في الماء مثل الأسماك؟
> **لماذا هذا مهم؟** لأن الجمع بين عدة سمات (وليس سمة واحدة كـ "يعيش في الماء") هو ما يعطي القوة التمييزية لـ Naïve Bayes — سمة `Give Birth=yes` كانت حاسمة هنا.

#### 🤔 تفعيل الفهم (اسأل نفسك):
> **سؤال:** ماذا يحدث للنتيجة لو كانت إحدى الاحتمالات الشرطية = 0 (مثل عدم وجود أي حالة `Give Birth=yes` بين non-mammals)؟
> **لماذا هذا مهم؟** لأن الناتج الكلي سيصبح صفراً بغض النظر عن باقي السمات — وهذه أهم نقطة ضعف عملية في Naïve Bayes تُحل بـ `Laplace Smoothing`.

#### 🤔 تفعيل الفهم (اسأل نفسك):
> **سؤال:** لو أضفنا سمة خامسة مترابطة تماماً مع "Live in Water" (مثل "Has Fins")، هل ستتغير جودة التصنيف؟
> **لماذا هذا مهم؟** لأن هذا يوضح حدود افتراض الاستقلالية: تكرار المعلومة نفسها عبر سمتين مترابطتين "يُضاعف تأثيرها" بشكل مصطنع في القرار النهائي.

## الجزء الثاني: ملخص منظم

### أهم التعاريف والمفاهيم
| المصطلح | التعريف | مثال/ملاحظة |
| --- | --- | --- |
| `Bayes' Theorem` | معادلة لحساب الاحتمال البعدي من القبلي | P(H\|X)=P(X\|H)P(H)/P(X) |
| `Posterior Probability` | احتمال الفرضية بعد رؤية البيانات | P(H\|X) |
| `Prior Probability` | احتمال الفرضية قبل رؤية البيانات | P(H) أو P(c) |
| `Naïve Bayes Classifier` | مصنف يفترض استقلالية شرطية بين السمات | يُستخدم في تصنيف النصوص |
| `Class-Conditional Independence` | استقلال قيم السمات عن بعضها بشرط الصنف | أساس تبسيط P(X\|c) |
| `Maximum Likelihood Classification` | اختيار الصنف الأعلى احتمالاً | argmax P(c\|X) |
| `Gaussian Distribution` | توزيع احتمالي جرسي الشكل للسمات المستمرة | يستخدم μ و σ |
| `Discretization` | تحويل سمة مستمرة إلى فئات | بديل لتقدير الكثافة |
| `Bayesian Belief Network` | نموذج بياني ينمذج الترابطات بين السمات | عكس الافتراض الساذج |

### المكونات الرئيسية (مرجع سريع)
| الأداة | الوظيفة | ملاحظة |
| --- | --- | --- |
| `P(c)` | الاحتمال القبلي للصنف | = عدد سجلات الصنف/كل السجلات |
| `P(Ai\|c)` (فئوية) | احتمال شرطي بالعدّ | = \|Ai,c\| / \|Dc\| |
| `P(Ai\|c)` (مستمرة) | احتمال شرطي بدالة الكثافة | = g(x, μc, σc) |
| `argmax` | اختيار القيمة العظمى | يحدد الصنف النهائي |

### جداول مقارنات سريعة
| المقارنة | Decision Tree | Naïve Bayes |
| --- | --- | --- |
| الأساس | تقسيمات متتالية (Entropy/Gini) | احتمالات (Bayes' Theorem) |
| المخرج | قرار قاطع | احتمال لكل صنف |
| التعامل مع السمات المترابطة | جيد (لا يفترض استقلالية) | ضعيف (يفترض الاستقلالية) |
| السرعة | أبطأ نسبياً في البناء | سريع جداً في التدريب والتنبؤ |

### قاموس المصطلحات
| الفئة | المصطلحات |
| --- | --- |
| احتمالات أساسية | `Prior`، `Posterior`، `Likelihood`، `Evidence` |
| مصنفات بايزية | `Naïve Bayes`، `Bayesian Belief Networks` |
| توزيعات | `Gaussian Distribution`، `Probability Density Estimation` |
| معالجة سمات | `Discretization`، `Categorical`، `Continuous-valued` |

### أبرز النقاط الذهبية
1. `P(X)` ثابتة عند المقارنة بين الأصناف، لذا يمكن تجاهلها في `argmax`.
2. الافتراض الساذج (استقلالية السمات) هو ما يجعل الحساب ممكناً وسريعاً رغم كونه غير واقعي تماماً.
3. أي احتمال شرطي = صفر يُصفّر الناتج الكلي — الحل هو `Laplace Smoothing`.
4. السمات الفئوية تُحسب بالعدّ المباشر، والمستمرة بدالة كثافة احتمالية (غالباً Gaussian).
5. الصنف النهائي = الصنف صاحب أعلى قيمة لـ `P(c) * ΠP(Ai|c)`.

### الأخطاء الشائعة عند الطلاب ⚠️
| الخطأ | التصحيح |
| --- | --- |
| الخلط بين P(H\|X) وP(X\|H) | تذكّر: الأول هو ما نريد إيجاده (بعد المشاهدة)، الثاني معطى تقريباً من البيانات |
| نسيان ضرب P(c) بعد حساب P(X\|c) | التصنيف النهائي يحتاج ضرب الاثنين معاً وليس P(X\|c) فقط |
| استخدام \|D\| بدل \|Dc\| في حساب P(Ai\|c) | يجب القسمة على عدد سجلات الصنف فقط وليس كل البيانات |
| تجاهل مشكلة الصفر في الاحتمالات | يجب استخدام Laplace Smoothing عند وجود احتمال = 0 |

### خطوات وإجراءات المحاضرة

#### ⚙️ الخطوات / الخوارزمية: تطبيق Bayes' Theorem لحساب احتمال فرضية
> **الهدف:** حساب الاحتمال البعدي لفرضية معينة بناءً على بيانات ملاحظة.
```algorithm
1 | تحديد الفرضية H والبيانات X | تعريف المسألة | تحديد ما نريد إيجاد احتماله
2 | تقدير P(H) | من البيانات التاريخية | نسبة انتشار الفرضية عموماً
3 | تقدير P(X|H) | من البيانات التاريخية | احتمال المشاهدة بشرط صحة الفرضية
4 | تقدير P(X) | من البيانات التاريخية | احتمال المشاهدة بشكل عام
5 | تطبيق المعادلة | القسمة والضرب | P(H|X) = P(X|H)*P(H)/P(X)
```
#### نقاط التنفيذ:
- إذا كان `P(X)` غير معروف مباشرة، يمكن حسابه كمجموع `P(X|H)*P(H)` عبر كل الفرضيات الممكنة.

#### ⚙️ الخطوات / الخوارزمية: Naïve Bayes Classification (مفصّلة سابقاً في §4.3)
> **الهدف:** تصنيف سجل جديد بناءً على أعلى احتمال بعدي مقدّر بافتراض الاستقلالية.
```algorithm
1 | حساب P(c) لكل صنف | عدّ على D | نسبة سجلات كل صنف
2 | حساب P(Ai|c) لكل سمة فئوية | عدّ شرطي | |Ai,c|/|Dc|
3 | حساب P(Ai|c) لكل سمة مستمرة | Gaussian | تقدير μ, σ ثم g(x,μ,σ)
4 | حساب Score(c) لكل صنف | ضرب تراكمي | P(c) * product P(Ai|c)
5 | اختيار argmax | مقارنة | الصنف الأعلى Score هو التصنيف النهائي
```
#### نقاط التنفيذ:
- كرر الحساب لكل صنف على حدة قبل المقارنة النهائية.
- لا تنسَ التحقق من مشكلة الصفر قبل الضرب.

### أنماط الأكواد والبنى المتكررة
| النمط | البنية الأساسية | متى تستخدمه |
| --- | --- | --- |
| العدّ الشرطي | `df[df['class']==c][df['A']==a].shape[0] / df[df['class']==c].shape[0]` | حساب P(Ai\|c) للسمات الفئوية |
| تقدير Gaussian | `scipy.stats.norm(mean, std).pdf(x)` | حساب P(Ai\|c) للسمات المستمرة |
| حاصل الضرب التراكمي | `np.prod([...])` | حساب P(X\|c) الكلي |
| argmax على القاموس | `max(scores, key=scores.get)` | اختيار الصنف النهائي |

### أنماط التعامل والسلوك
| السيناريو | التعامل الصحيح | لماذا؟ |
| --- | --- | --- |
| احتمال شرطي = 0 | تطبيق Laplace Smoothing | لتفادي تصفير الناتج بالكامل |
| سمة مستمرة بتوزيع غير طبيعي | استخدام Discretization بدل Gaussian | Gaussian تفترض توزيعاً جرسياً فقط |
| عدد كبير من السمات | استخدام Naïve Bayes بدل الطرق الأثقل حسابياً | لأنها تتوسع خطياً مع عدد السمات |

### الأفكار الرئيسية الشاملة
النقطة المحورية في هذه المحاضرة هي أن `Naïve Bayes` تحوّل مشكلة تصنيف معقدة إلى سلسلة من عمليات ضرب بسيطة، بفضل افتراض الاستقلالية الشرطية. الثمن المدفوع هو فقدان بعض الدقة عند وجود ترابط حقيقي بين السمات، لكن المكسب هو سرعة حساب هائلة وحاجة أقل لبيانات تدريب ضخمة — وهذا ما يجعلها الخيار الأول في تطبيقات مثل تصفية البريد المزعج وتحليل المشاعر.

### السؤال 5 (hard) — سيناريو كود
```python
def naive_bayes_score(x, class_probs, cond_probs):
    score = 1  # (1)
    for attr in x:
        score *= cond_probs[attr]
    return score
```
ما الخطأ في هذا الكود بالنسبة لحساب `Score(c)` الصحيح؟
أ) لا يوجد خطأ  ب) نسي ضرب `class_probs` (أي P(c)) في السطر (1) أو النتيجة النهائية  ج) استخدام `*=` خطأ ويجب استخدام `+=`  د) القيمة الابتدائية يجب أن تكون 0
**الإجابة الصحيحة: ب**
**التعليل:** المعادلة الصحيحة `Score(c) = P(c) * Π P(Ai|c)`، والكود يحسب فقط حاصل الضرب دون احتساب P(c)؛ ج خاطئ لأن الضرب هو العملية الصحيحة، ود خاطئ لأن القيمة الابتدائية للضرب يجب أن تكون 1 لا 0.

### سؤال تصحيح 5 — logic
**الكود التالي يحتوي خطأ:**
```python
def naive_bayes_predict(x, classes, prior, cond_probs):
    scores = {}
    for c in classes:
        score = 1  # (1) بداية بدون احتساب prior
        for attr, val in x.items():
            score *= cond_probs[c][attr].get(val, 1)  # (2) القيمة الافتراضية خطأ
        scores[c] = score
    return max(scores, key=scores.get)
```
**اكتشف الخطأ:** خطآن: (1) لم يُدمج `prior[c]` في `score` الابتدائي، و(2) القيمة الافتراضية عند غياب المفتاح هي `1` بدلاً من `0`، مما يجعل السمات غير الموجودة في بيانات التدريب "تُهمَل" بدل أن تُصفّر الناتج (وهذا في الواقع سلوك خاطئ لأنه يخفي مشكلة البيانات الناقصة دون معالجة صريحة).

**التصحيح:**
```python
def naive_bayes_predict(x, classes, prior, cond_probs):
    scores = {}
    for c in classes:
        score = prior[c]
        for attr, val in x.items():
            score *= cond_probs[c][attr].get(val, 0)
        scores[c] = score
    return max(scores, key=scores.get)
```
**شرح الحل:**
1. يجب أن يبدأ `score` بقيمة `P(c)` وليس 1 فقط، لأن المعادلة الكاملة هي `P(c) * Π P(Ai|c)`.
2. القيمة الافتراضية الصحيحة عند غياب قيمة معينة في بيانات التدريب هي 0 (تعبيراً عن عدم الملاحظة)، مع التنبيه لضرورة استخدام Laplace Smoothing عملياً لتفادي تصفير الناتج بالكامل.
3. هذا يوضح أهمية معالجة الحالات الحدّية (Edge Cases) في أي تطبيق فعلي للخوارزمية.

## الجزء الخامس: أسئلة نظرية متوقعة بالامتحان

### السؤال 1: ما الفرق بين الاحتمال القبلي (Prior) والاحتمال البعدي (Posterior)؟
**نموذج الإجابة:** 1. التعريف: القبلي `P(H)` معروف قبل رؤية أي بيانات، والبعدي `P(H|X)` محسوب بعد رؤية البيانات X. 2. المكونات: يرتبطان عبر نظرية بايز والاحتمال الشرطي `P(X|H)`. 3. مثال: احتمال إصابة أي شخص بمرض نادر (Prior) مقابل احتمال إصابته بعد ظهور أعراض معينة (Posterior). 4. يُستخدم القبلي كنقطة بداية، والبعدي كنتيجة نهائية للتصنيف.

---

### السؤال 2: لماذا يمكن حذف P(X) من معادلة argmax في Naïve Bayes؟
**نموذج الإجابة:** 1. التعريف: P(X) هو احتمال ملاحظة البيانات بغض النظر عن الصنف. 2. الشرط: يبقى ثابتاً عبر كل الأصناف عند مقارنتها لنفس السجل X. 3. مثال: عند مقارنة yes وno لنفس السجل، P(X) يظهر في مقام كلا الطرفين فلا يغيّر أيّهما أكبر. 4. يُستخدم هذا التبسيط لتسريع الحساب دون التأثير على النتيجة.

---

### السؤال 3: صف كيفية حساب P(Ai|c) عندما تكون Ai سمة مستمرة.
**نموذج الإجابة:** 1. التعريف: نستخدم إما Discretization أو تقدير كثافة احتمالية (غالباً Gaussian). 2. المكونات: نحتاج تقدير μ وσ من بيانات التدريب لكل صنف. 3. مثال: P(Income=120|No) محسوبة بدالة g(x,μ,σ) بقيمة 0.0072. 4. تُستخدم عندما لا يمكن العدّ المباشر بسبب استمرارية القيم.

---

### السؤال 4: اشرح مشكلة "الاحتمال الصفري" وحلها.
**نموذج الإجابة:** 1. التعريف: عندما لا يظهر تركيب معين (سمة+صنف) في بيانات التدريب، يصبح P(Ai|c)=0. 2. الشرط: هذا يُصفّر حاصل الضرب الكلي P(X|c) بغض النظر عن باقي السمات. 3. مثال: P(Refund=Yes|Yes)=0 في جدول Evade. 4. الحل: `Laplace Smoothing` بإضافة عدد صغير للبسط والمقام لتفادي الصفر التام.

---

### السؤال 5: ما هي الفرضية الأساسية التي يعتمد عليها Naïve Bayes وما مخاطرها؟
**نموذج الإجابة:** 1. التعريف: افتراض class-conditional independence بين السمات. 2. الشرط: يفترض أن معرفة قيمة سمة لا تخبرنا شيئاً عن سمة أخرى بشرط معرفة الصنف. 3. مثال: افتراض استقلال Give Birth وLive in Water رغم ترابطهما الفعلي بيولوجياً. 4. المخاطر: قد يُبالغ في تأثير السمات المترابطة على القرار النهائي.

---

### السؤال 6: كيف يُستخدم Naïve Bayes في تصنيف النصوص عملياً؟
**نموذج الإجابة:** 1. التعريف: كل كلمة تُعامل كسمة فئوية (موجودة/غير موجودة أو تكرارها). 2. المكونات: نحسب P(كلمة|صنف) لكل كلمة في القاموس بشرط كل صنف (مثل Spam/Not-Spam). 3. مثال: P("Free"|Spam)=0.7. 4. متى نستخدمه: عندما يكون عدد السمات (الكلمات) كبيراً جداً وتحتاج طريقة سريعة الحساب.

---

### السؤال 7: ما الفرق بين Naïve Bayes وBayesian Belief Networks من حيث التمثيل؟
**نموذج الإجابة:** 1. التعريف: Naïve Bayes يفترض استقلالية كاملة، بينما Bayesian Belief Networks تمثل الترابطات بشكل بياني (Graph). 2. المكونات: BBN تحتوي عقداً وحواف تمثل الاعتماديات الشرطية بين السمات. 3. مثال: BBN يمكنها نمذجة أن "التدخين" يؤثر على "السرطان" و"السعال" معاً. 4. متى نستخدم كلاً منهما: Naïve Bayes للسرعة، BBN للدقة عند وجود ترابطات معروفة.

---

### السؤال 8: كيف نفسّر ناتج دالة Gaussian عندما تكون القيمة x بعيدة جداً عن μ؟
**نموذج الإجابة:** 1. التعريف: دالة Gaussian تعطي قيمة كثافة أعلى قرب μ وتتناقص كلما ابتعدنا عنه. 2. الشرط: هذا بسبب الأس السالب `-(x-μ)²/2σ²` الذي يكبر (بالقيمة المطلقة) كلما ابتعدت x عن μ فيصغّر الأس الأسّي. 3. مثال: احتمال دخل بعيد جداً عن المتوسط يكون قريباً من الصفر. 4. يُستخدم هذا لتمييز القيم النموذجية عن القيم الشاذة (Outliers) لصنف معين.

---

### السؤال 9: علّل لماذا يُعتبر Naïve Bayes مناسباً للبيانات عالية الأبعاد (High-dimensional).
**نموذج الإجابة:** 1. التعريف: الحساب يتوسع خطياً (Linear) مع عدد السمات وليس أسّياً. 2. الشرط: بفضل افتراض الاستقلالية، لا نحتاج لتقدير احتمالات مشتركة لكل تركيبة من السمات معاً. 3. مثال: في تصنيف نص بآلاف الكلمات، لا يمكن عملياً حساب احتمال مشترك لكل التركيبات، لكن يمكن حساب كل كلمة على حدة. 4. متى نستخدمه: عند عدد سمات كبير جداً مع بيانات تدريب محدودة نسبياً.

---

### السؤال 10: صف خطوات تصنيف سجل جديد بواسطة Naïve Bayes من البداية للنهاية.
**نموذج الإجابة:** 1. التعريف: عملية تتضمن حساب الاحتمالات القبلية والشرطية ثم المقارنة. 2. المكونات: P(c) لكل صنف، P(Ai|c) لكل سمة (فئوية أو مستمرة)، ثم الضرب التراكمي. 3. مثال: مثال buys_computer الكامل الموضح في القسم 5. 4. الخطوة الأخيرة: اختيار argmax لأعلى قيمة Score(c) كالتصنيف النهائي.

| الموضوع | النقاط |
| --- | --- |

| التبسيط | يمكن حذف P(X) عند المقارنة بين الأصناف لأنها ثابتة |
| الاستقلالية | P(X\|c) = Π P(Ai\|c) بافتراض class-conditional independence |
| السمات الفئوية | P(Ai\|c) = \|Ai,c\| / \|Dc\| |
| السمات المستمرة | غالباً عبر Gaussian: g(x,μ,σ) |
| مشكلة الصفر | تُحل بـ Laplace Smoothing |

### 🔑 مرجع سريع
| الرمز/المصطلح | المعنى | يُستخدم في |
| --- | --- | --- |
| `H` | الفرضية (انتماء لصنف) | Bayes' Theorem |
| `X` | سجل بيانات غير معروف الصنف | كل معادلات الفصل |
| `P(c)` | الاحتمال القبلي للصنف | حساب Score النهائي |
| `argmax` | اختيار القيمة العظمى | التصنيف النهائي |
| `μ, σ` | متوسط وانحراف معياري | Gaussian Density |

### 🔑 قواعد ذهبية لا تُنسى
| # | القاعدة |
| --- | --- |
| 1 | التصنيف النهائي = argmax عبر [P(c) * Π P(Ai\|c)] |
| 2 | القسمة في P(Ai\|c) دائماً على \|Dc\| وليس \|D\| |

```python
import numpy as np
import pandas as pd

# === Data Preprocessing ===
# Load or construct the training DataFrame (columns = attributes + class label)
# df = pd.read_csv("training_data.csv")

# === Naive Bayes: Prior Probability ===
def prior_prob(df, class_col, c):
    total = len(df)                       # |D|
    count_c = len(df[df[class_col] == c]) # |c|
    return count_c / total                # P(c) = |c| / |D|

# === Naive Bayes: Conditional Probability (Categorical Attribute) ===
def cond_prob_categorical(df, attr, val, class_col, c):
    class_df = df[df[class_col] == c]           # subset of class c, |Dc|
    match = class_df[class_df[attr] == val]      # |Ai,c|
    if len(class_df) == 0:
        return 0
    return len(match) / len(class_df)            # P(Ai|c) = |Ai,c| / |Dc|

# === Naive Bayes: Conditional Probability (Continuous Attribute, Gaussian) ===
def gaussian_pdf(x, mean, std):
    if std == 0:
        return 1                                 # avoid division by zero
    coeff = 1 / (np.sqrt(2 * np.pi) * std)         # normalizing coefficient
    exponent = np.exp(-((x - mean) ** 2) / (2 * std ** 2))  # Gaussian exponent
    return coeff * exponent                        # g(x, mean, std)

def cond_prob_continuous(df, attr, x_value, class_col, c):
    class_df = df[df[class_col] == c]
    mean = class_df[attr].mean()                   # mu_c
    std = class_df[attr].std()                      # sigma_c
    return gaussian_pdf(x_value, mean, std)

# === Naive Bayes: Full Classification ===
def naive_bayes_classify(df, class_col, tuple_x, continuous_attrs=None):
    if continuous_attrs is None:
        continuous_attrs = []
    classes = df[class_col].unique()
    scores = {}
    for c in classes:
        score = prior_prob(df, class_col, c)        # start with P(c)
        for attr, val in tuple_x.items():
            if attr in continuous_attrs:
                p = cond_prob_continuous(df, attr, val, class_col, c)
            else:
                p = cond_prob_categorical(df, attr, val, class_col, c)
            score *= p                                # multiply P(Ai|c)
        scores[c] = score
    predicted_class = max(scores, key=scores.get)     # argmax
    return predicted_class, scores
```
