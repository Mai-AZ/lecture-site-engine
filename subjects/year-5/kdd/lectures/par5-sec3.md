## الجزء الأول: الشرح التفصيلي (سطر بسطر / فقرة بفقرة)

### 1. دوال أكثر تعقيدًا لمدخل واحد (More Complex Functions of a Single Input)

#### النص الأصلي يقول:
> Model (More generally…): yᵢ = w₀h₀(xᵢ) + w₁h₁(xᵢ) + … + w_D h_D(xᵢ) + εᵢ = Σ_{j=0}^{D} wⱼhⱼ(xᵢ) + εᵢ
> feature 1 = h₀(x)… often 1 (constant); feature 2 = h₁(x)… e.g., x; feature 3 = h₂(x)… e.g., x² or sin(2πx/12); … feature D+1 = h_D(x)… e.g., xᵖ
> Note: Output of feature extractor is not x but features of the input h(x). Some set of functions of x.

#### الشرح المبسّط:
النموذج هنا لم يعد يفترض أن العلاقة بين `x` و`y` خط مستقيم فقط. بدلاً من ذلك، نحوّل المدخل `x` إلى مجموعة من "الميزات" (`features`) عبر دوال `h₀(x), h₁(x), ..., h_D(x)`، ثم نضرب كل ميزة بوزن (`weight`) خاص بها ونجمعها. مثلاً بدل الاكتفاء بـ `x`، يمكن أن نضيف `x²` أو `sin(x)` كميزة إضافية، فيصبح النموذج قادرًا على تمثيل منحنيات وليس خطوطًا فقط، رغم أن العلاقة تبقى **خطية بالنسبة للأوزان** `w`.

**لماذا؟** لأن كثيرًا من الظواهر الحقيقية (كسعر المنزل مقابل المساحة) لا تتبع خطًا مستقيمًا تمامًا، فنحتاج مرونة أكبر في الشكل دون أن نفقد بساطة الحل الرياضي للانحدار الخطي.

#### 💡 التشبيه:
> تخيل أنك تصف طعم وجبة لصديقك، فبدل كلمة واحدة ("لذيذة") تستخدم عدة أوصاف: حلو، حامض، حار — كل وصف هو "ميزة" منفصلة تصف نفس المدخل (الوجبة) من زاوية مختلفة.
> **وجه الشبه:** الأوصاف المتعددة = الميزات `h₀(x), h₁(x), ..., h_D(x)` التي تصف نفس `x` من زوايا مختلفة.

#### 📐 المعادلة: النموذج العام متعدد الميزات لمدخل واحد

$$
y_i = \sum_{j=0}^{D} w_j h_j(x_i) + \varepsilon_i
$$

**الشرح:**
> `yᵢ` القيمة الفعلية للمخرج | `wⱼ` معامل الانحدار (الوزن) للميزة رقم `j` | `hⱼ(xᵢ)` الميزة رقم `j` المُشتقة من المدخل `xᵢ` (دالة على `x`، مثل `x²` أو `sin(x)`) | `εᵢ` حد الخطأ العشوائي (`noise`) | `D` أعلى رتبة ميزة مستخدمة

🤔 **تفعيل الفهم (اسأل نفسك):**
> **سؤال:** إذا كانت `h₀(x)=1`، `h₁(x)=x`، `h₂(x)=x²`، فما اسم هذا النوع من الانحدار؟
> **لماذا هذا مهم؟** لأنه يُظهر أن `Polynomial Regression` هو مجرد حالة خاصة من هذا النموذج العام عندما تكون الميزات هي قوى `x` المتتالية.

### 2. المدخلات المتعددة (Multiple Inputs)

#### النص الأصلي يقول:
> Even if we have a more complicated functions of a single input, it still might not be a great predictive model. "Only 1 bathroom! Not same as my 3 bathrooms." Many Possible inputs can be added to the regression model (Square feet, #bathrooms, # bedrooms, Year built, …).

#### الشرح المبسّط:
حتى لو جعلنا الدالة معقدة جدًا (منحنيات، أسس عالية)، فإن الاعتماد على مدخل واحد فقط (مثل المساحة بالقدم المربع) يبقى قاصرًا، لأن هناك عوامل أخرى تؤثر في السعر لا علاقة لها بالمساحة، مثل عدد الحمامات أو سنة البناء. منزلان بنفس المساحة قد يختلف سعرهما تمامًا بسبب عدد الحمامات.

**لماذا؟** لأن الواقع متعدد الأبعاد؛ متغير واحد نادرًا ما يكفي لتفسير ظاهرة معقدة، لذا نحتاج لتوسيع النموذج ليأخذ عدة مدخلات معًا.

#### 💡 التشبيه:
> تقييم طالب بناءً على درجة اختبار واحد فقط ظالم؛ الأصح أن تنظر أيضًا لحضوره ومشاركته وواجباته.
> **وجه الشبه:** درجة الاختبار الواحدة = مدخل واحد (المساحة) | مجموع (اختبار + حضور + واجبات) = مدخلات متعددة (مساحة + حمامات + غرف نوم).

---

### 2.1. النموذج الهندسي: Simple Hyperplane

#### النص الأصلي يقول:
> In this multiple dimensional space, we can fit some Simple hyperplane that models the relationship between the inputs and the output.
> Model: Inputs: x = (x[1], x[2], …, x[d]) — d-dim vector
> yᵢ = w₀ + w₁xᵢ[1] + … + w_d xᵢ[d] + εᵢ
> feature 1 = 1 ; feature 2 = x[1] … e.g., sq. ft. ; feature 3 = x[2] … e.g., #bath ; … feature d+1 = x[d] … e.g., lot size

#### الشرح المبسّط:
بدل أن يكون المدخل رقمًا واحدًا `x`، يصبح متجهًا `x` بأبعاد `d` (مساحة، عدد حمامات، عدد غرف...). النموذج الآن لا يرسم خطًا مستقيمًا في مستوى ثنائي الأبعاد، بل "مستوى مسطح" (`hyperplane`) في فضاء متعدد الأبعاد. لكل بُعد وزن `wⱼ` خاص به.

**لماذا؟** لأن كل بُعد إضافي (كل مدخل جديد) يمثل محورًا جديدًا في الفضاء الهندسي؛ فعندما يكون لدينا مدخلان (مساحة وعدد حمامات) يصبح الرسم ثلاثي الأبعاد، والنموذج فيه هو مستوٍ مسطح (وليس خطًا).

#### 💡 التشبيه:
> رسم خط على ورقة (بعدين: x وy) سهل، لكن تخيل أنك تريد رسم "سطح" مستوٍ داخل صندوق ثلاثي الأبعاد — هذا هو الـ hyperplane.
> **وجه الشبه:** الخط في بعدين = Simple Linear Regression | المستوى في أبعاد أكثر = Multiple Regression الذي يُشكّل hyperplane.

📐 **المعادلة: نموذج الانحدار المتعدد المدخلات**

$$
y_i = w_0 + w_1 x_i[1] + w_2 x_i[2] + \dots + w_d x_i[d] + \varepsilon_i
$$

**الشرح:**
> `x = (x[1], x[2], ..., x[d])` متجه المدخلات بأبعاد `d` | `x[j]` المدخل رقم `j` (قيمة عددية واحدة) | `w₀` الحد الثابت (Intercept) | `wⱼ` وزن المدخل رقم `j`

### 3. الصورة الأعم: منحنى بأبعاد D (D-dimensional curve)

#### النص الأصلي يقول:
> More generically, a D-dimensional curve. Model: yᵢ = w₀h₀(xᵢ) + w₁h₁(xᵢ) + … + w_D h_D(xᵢ) + εᵢ = Σ_{j=0}^{D} wⱼhⱼ(xᵢ) + εᵢ
> x[j] = jᵗʰ input (scalar), hⱼ(x) = jᵗʰ feature (scalar), xᵢ = input of iᵗʰ data point (vector), xᵢ[j] = jᵗʰ input of iᵗʰ data point (scalar)
> feature 1 = h₀(x) … e.g., 1 ; feature 2 = h₁(x) … e.g., x[1] = sq. ft. ; feature 3 = h₂(x) … e.g., x[2] = #bath — or, log(x[7]) x[2] = log(#bed) × #bath ; … feature D+1 = h_D(x) … some other function of x[1],…,x[d]

#### الشرح المبسّط:
هذه هي الصيغة النهائية الأكثر عمومية: تجمع بين فكرتَي "دوال معقدة لمدخل واحد" (البند 1) و"مدخلات متعددة" (البند 2). الآن كل ميزة `hⱼ(x)` يمكن أن تكون دالة على **متجه** كامل من المدخلات، ليس بالضرورة `x[j]` وحده، بل قد تكون تركيبة مثل `log(x[7]) × x[2]` (تفاعل بين متغيرين).

**لماذا؟** لأن الظواهر الحقيقية غالبًا فيها تفاعلات معقدة بين المتغيرات (مثلاً تأثير عدد غرف النوم على السعر يختلف حسب عدد الحمامات)، وصيغة الميزات العامة `hⱼ(x)` تسمح بالتعبير عن ذلك رياضيًا مع بقاء النموذج **خطيًا في الأوزان** `w`.

#### ⚖️ المقايضة: نموذج بسيط مقابل نموذج معقّد

| | نموذج بسيط (ميزات قليلة) | نموذج معقّد (ميزات كثيرة/تفاعلات) |
| --- | --- | --- |
| المزايا | سهل الفهم، أقل عرضة لـ `Overfitting`، سريع التدريب | يلتقط علاقات دقيقة، دقة أعلى على بيانات التدريب |
| العيوب | قد لا يلتقط العلاقات الحقيقية (`Underfitting`) | خطر `Overfitting`، صعب التفسير، بطيء |
| متى تختاره | بيانات قليلة أو علاقة بسيطة معروفة | بيانات كثيرة وعلاقة معقدة موثقة |

### 4. تقييم الأداء (Assessing Performance)

#### النص الأصلي يقول:
> "Remember that all models are wrong; the practical question is how wrong do they have to be to not be useful." George Box, 1987.
> due to inaccurate listing price, how much am I losing $? E.g., Too high → few lookers + no/low offers. My predictions: Loss = ???
> Loss function: L(y, f_ŵ(x)) — "Cost of using ŵ at x when y is true"

#### الشرح المبسّط:
مهما كان النموذج جيدًا فهو تقريب وليس الحقيقة المطلقة (مقولة جورج بوكس الشهيرة). السؤال العملي ليس "هل النموذج مثالي؟" بل "كم أخسر فعليًا بسبب خطأ التنبؤ؟". لتحويل هذا السؤال إلى رقم رياضي نستخدم **دالة الخسارة** `Loss Function`، والتي تقيس تكلفة الفرق بين التوقع `f_ŵ(x)` والقيمة الفعلية `y`.

**لماذا؟** لأننا نحتاج مقياسًا رقميًا موحدًا لمقارنة النماذج المختلفة واختيار أفضلها، بدل الحكم الانطباعي.

📐 **المعادلة: دالة الخسارة العامة**

$$
L(y, f_{\hat{w}}(x))
$$

**الشرح:**
> `y` القيمة الفعلية (`actual value`) | `f_ŵ(x)` القيمة المتوقعة `ŷ` باستخدام الأوزان المقدَّرة `ŵ` | الناتج هو "تكلفة" استخدام `ŵ` عند نقطة `x` بينما القيمة الحقيقية هي `y`

#### 🤔 تفعيل الفهم (اسأل نفسك):
> **سؤال:** لماذا لا نكتفي بالقول "النموذج دقيق 95%" دون استخدام Loss Function رقمية؟
> **لماذا هذا مهم؟** لأن نسبة الدقة وحدها لا تخبرنا بحجم الخسارة المالية الفعلية أو نوع الخطأ (تقدير أعلى أم أقل من السعر الحقيقي)، بينما Loss Function تسمح بربط الخطأ بتكلفة عملية حقيقية (مثل خسارة بيع منزل).

### 5. خطأ التدريب (Training Error)

#### النص الأصلي يقول:
> 1. Define a loss function L(y, f_ŵ(x)) — E.g., squared error, absolute error, …
> 2. Training error = avg. loss on houses in training set = (1/N) Σ_{i=1}^{N} L(yᵢ, f_ŵ(xᵢ)) — "fit using training data"
> ŵ minimizes RSS of training data

#### الشرح المبسّط:
بعد اختيار دالة خسارة (مثل مربع الخطأ)، نحسب **متوسط** هذه الخسارة عبر كل نقاط بيانات **التدريب** فقط. هذا الرقم يسمى `Training Error`. المشكلة أن الأوزان `ŵ` نفسها تم اختيارها لتُصغّر هذا الخطأ (عبر تصغير `RSS` — Residual Sum of Squares)، وبالتالي القيمة متفائلة بشكل غير واقعي.

**لماذا؟** لأن قياس أداء النموذج على نفس البيانات التي "درسها" أشبه بتصحيح امتحان الطالب من نفس الأسئلة التي حفظ حلولها — لا يعكس فهمه الحقيقي.

📐 **المعادلة: خطأ التدريب**

$$
\text{Training error}(\hat{w}) = \frac{1}{N}\sum_{i=1}^{N} L\big(y_i, f_{\hat{w}}(x_i)\big)
$$

**الشرح:**
> `N` عدد نقاط بيانات التدريب | `L(yᵢ, f_ŵ(xᵢ))` الخسارة عند نقطة `i` | الأوزان `ŵ` تم اختيارها خصيصًا لتُصغّر هذا المجموع (`RSS`) على بيانات التدريب نفسها

**الفهم الخاطئ الشائع ❌:** خطأ تدريب منخفض جدًا يعني أن النموذج ممتاز ويمكن الاعتماد عليه في التنبؤ بأي بيانات جديدة.
**الفهم الصحيح ✅:** خطأ تدريب منخفض قد يعني فقط أن النموذج "حفظ" بيانات التدريب (`Overfitting`)؛ الحكم الحقيقي يأتي من أداء النموذج على بيانات لم يرها من قبل.

### 6. خطأ التدريب مقابل تعقيد النموذج (Training error vs. model complexity)

#### النص الأصلي يقول:
> Training error is overly optimistic (because ŵ was fit to training data). Small training error ≠> good predictions — unless training data includes everything you might ever see.

#### الشرح المبسّط:
كلما زاد تعقيد النموذج (زاد عدد الميزات `D` أو رتبة كثير الحدود)، انخفض خطأ التدريب باستمرار حتى يصل قريبًا من الصفر إذا كان النموذج معقدًا بما يكفي ليمر بكل نقطة بيانات تقريبًا. لكن هذا لا يعني أن النموذج سيتنبأ جيدًا ببيانات جديدة.

**لماذا؟** لأن النموذج المعقد جدًا يبدأ "بحفظ" الضوضاء (`noise`) الموجودة في بيانات التدريب بدلاً من تعلّم النمط الحقيقي وراءها.

#### ⚙️ الخطوات / الخوارزمية: العلاقة بين تعقيد النموذج وخطأ التدريب

#### ما هدف هذه العملية؟
> توضيح لماذا لا يصلح خطأ التدريب وحده كمقياس لاختيار أفضل نموذج.

```algorithm
1 | ابدأ بنموذج بسيط جدًا (ثابت فقط) | حساب Training Error | خطأ مرتفع (Underfitting)
2 | زد التعقيد تدريجيًا (خط، ثم منحنى تربيعي) | حساب Training Error | الخطأ يقل تدريجيًا
3 | استمر بزيادة التعقيد (كثير حدود من رتبة عالية) | حساب Training Error | الخطأ يقترب من الصفر
4 | لاحظ أن المنحنى النهائي "يهتز" لملامسة كل نقطة | مقارنة بصرية | إشارة إلى Overfitting
```

#### نقاط التنفيذ:
- خطأ التدريب دالة **تناقصية دائمًا** مع زيادة التعقيد، ولا يمكن الاعتماد عليها وحدها.
- الاستثناء الوحيد الذي يجعل خطأ التدريب موثوقًا هو أن تكون بيانات التدريب تغطي كل الحالات الممكنة فعليًا — وهذا نادرًا ما يتحقق في الواقع.

### 7. خطأ الاختبار (Test Error)

#### النص الأصلي يقول:
> Approximating generalization error by looking at houses not in training set. Use squared error loss (y − f_ŵ(x))².
> Test error(ŵ) = 1/N * [(\$_test1 − f_ŵ(sq.ft._test1))² + (\$_test2 − f_ŵ(sq.ft._test2))² + (\$_test3 − f_ŵ(sq.ft._test3))² + … include all test houses]

#### الشرح المبسّط:
لتفادي التفاؤل الزائف لخطأ التدريب، نحتفظ بمجموعة بيانات منفصلة تمامًا لم يرها النموذج أثناء التدريب، تسمى **مجموعة الاختبار** (`Test Set`)، ونحسب متوسط الخسارة عليها. هذا الرقم يقارب فعليًا **خطأ التعميم** (`Generalization Error`) — أي كيف سيتصرف النموذج مع بيانات حقيقية جديدة.

**لماذا؟** لأن الهدف النهائي للنموذج ليس تفسير بيانات الماضي، بل التنبؤ الصحيح ببيانات المستقبل التي لم يرها بعد.

📐 **المعادلة: خطأ الاختبار (مربع الخطأ)**

$$
\text{Test error}(\hat{w}) = \frac{1}{N}\sum_{i=1}^{N} \big(y_{\text{test},i} - f_{\hat{w}}(x_{\text{test},i})\big)^2
$$

**الشرح:**
> `y_test,i` القيمة الفعلية لبيت اختبار رقم `i` | `f_ŵ(x_test,i)` تنبؤ النموذج (المُدرَّب على بيانات التدريب فقط) لنفس البيت | الفرق بينهما يُربّع ثم يُؤخذ متوسطه على كل بيانات الاختبار

#### 🔄 قبل / بعد: استخدام بيانات التدريب مقابل الاختبار

**قبل (Training Error):**
```python
# same data used for fitting AND evaluating -> optimistic
error = mean((y_train - model.predict(X_train))**2)
```

**بعد (Test Error):**
```python
# data unseen during fitting -> realistic estimate
error = mean((y_test - model.predict(X_test))**2)
```

**ماذا تغيّر؟** انتقلنا من قياس "كم يحفظ النموذج؟" إلى قياس "كم يعمّم النموذج على بيانات جديدة؟".

### 8. خطأ التدريب والاختبار والخطأ الحقيقي مقابل تعقيد النموذج

#### النص الأصلي يقول:
> Training, True, & Test Error Vs. Model Complexity — generalization (true) error, best error, training error.
> Overfitting if: If there exists a model with estimated params w' such that (1) training error(ŵ) < training error(w') and (2) true error(ŵ) > true error(w')

#### الشرح المبسّط:
عند رسم الأخطاء الثلاثة (تدريب، اختبار/حقيقي) مقابل تعقيد النموذج نلاحظ:
- **خطأ التدريب**: يتناقص باستمرار مع زيادة التعقيد.
- **الخطأ الحقيقي/خطأ الاختبار**: يتناقص في البداية ثم يبدأ بالارتفاع مجددًا بعد نقطة معينة (شكل حرف U).
نقطة أدنى الخطأ الحقيقي هي أفضل توازن. **التعريف الرسمي للـ Overfitting**: يحدث إفراط في التخصيص إذا وُجد نموذج آخر بأوزان `w'` أداؤه على التدريب أسوأ (`training error` أعلى) لكن أداءه الحقيقي أفضل (`true error` أقل) من النموذج الحالي `ŵ`.

**لماذا؟** لأن هذا التعريف الدقيق يفصل بوضوح بين "يبدو جيدًا على التدريب" و"جيد فعليًا" — فالنموذج المُفرط في التخصيص يخدع نفسه بخطأ تدريب منخفض بينما هو أسوأ في الواقع.

#### 💡 التشبيه:
> طالب يحفظ إجابات امتحان سابق حرفيًا (خطأ تدريب = صفر تقريبًا)، لكنه يفشل في امتحان جديد بأسئلة مشابهة لكن مختلفة الصياغة (خطأ حقيقي مرتفع).
> **وجه الشبه:** حفظ الإجابات = تصغير خطأ التدريب فقط | الفهم الحقيقي للمادة = تصغير الخطأ الحقيقي (التعميم).

📐 **المعادلة: تعريف Overfitting الرسمي**

$$
\text{Overfitting}(\hat{w}) \iff \exists\, w' : \text{training\_error}(\hat{w}) < \text{training\_error}(w') \ \text{و}\ \text{true\_error}(\hat{w}) > \text{true\_error}(w')
$$

**الشرح:**
> `ŵ` أوزان النموذج الحالي (المشتبه بأنه مُفرط في التخصيص) | `w'` أوزان نموذج بديل (أبسط عادة) | الشرطان معًا (١) أفضل على التدريب و(٢) أسوأ حقيقيًا هما ما يُثبتان الإفراط في التخصيص

#### 🛠️ استكشاف الأخطاء

| الخطأ | السبب | الحل |
| --- | --- | --- |
| اختيار النموذج بناءً على أقل خطأ تدريب فقط | الخلط بين خطأ التدريب والخطأ الحقيقي | استخدم مجموعة اختبار أو `Cross-Validation` لاختيار النموذج |
| نموذج بسيط جدًا يعطي خطأ مرتفع في كل الحالات | `Underfitting` — النموذج لا يملك مرونة كافية لالتقاط النمط | زد عدد الميزات/رتبة كثير الحدود بحذر |
| نموذج معقد جدًا يعطي خطأ اختبار مرتفع رغم خطأ تدريب شبه صفري | `Overfitting` — النموذج حفظ الضوضاء | قلّل التعقيد، استخدم `Regularization` (Ridge/Lasso)، أو زد بيانات التدريب |

### 9. توليد بيانات اصطناعية (Synthetic Data Generation)

#### النص الأصلي يقول:
> To illustrate how linear regression works, we first generate a random 1-dimensional vector, x, from a uniform distribution. The output variable y has a linear relationship with x according to the following equation: y = −3x + 1 + epsilon — where epsilon corresponds to random noise sampled from a Gaussian distribution with mean 0 and standard deviation of 1.

#### الشرح المبسّط:
لتعلّم كيف يعمل الانحدار الخطي عمليًا، يُنشئ المحاضر بيانات مصطنعة (`Synthetic Data`) بدل استخدام بيانات حقيقية معقدة، لأننا نعرف مسبقًا "الحقيقة الكاملة" (`y = -3x + 1`) ويمكننا مقارنة ما يتعلمه النموذج بهذه الحقيقة المعروفة. يُضاف ضجيج عشوائي (`epsilon`) لمحاكاة عدم دقة القياس في الواقع.

**لماذا؟** لأن استخدام بيانات معروفة الإجابة الصحيحة مسبقًا يسمح لنا بالتحقق: هل يستطيع النموذج فعلًا استرجاع الميل (`-3`) والمقطع (`1`) الحقيقيين من البيانات المشوشة؟

#### 💻 الكود: توليد البيانات ورسمها

#### ما هذا الكود؟
> يولّد 200 نقطة بيانات عشوائية `x` من توزيع منتظم بين 0 و1، يحسب `y` الحقيقية بالمعادلة الخطية، يضيف ضجيجًا غاوسيًا، ثم يرسم النقاط مع الخط الحقيقي.

```python
%matplotlib inline
import numpy as np                      # numerical operations
import matplotlib.pyplot as plt          # plotting

seed = 1                                 # seed for random number generation
numInstances = 200                       # number of data instances
np.random.seed(seed)                     # fix randomness for reproducibility
X = np.random.rand(numInstances, 1).reshape(-1, 1)   # X ~ Uniform(0,1), shape (200,1)
y_true = -3 * X + 1                      # true underlying linear function
y = y_true + np.random.normal(size=numInstances).reshape(-1, 1)  # add Gaussian noise

plt.scatter(X, y, color='black')                 # noisy observed data
plt.plot(X, y_true, color='blue', linewidth=3)    # true (noise-free) line
plt.title('True function: y = -3X + 1')
plt.xlabel('X')
plt.ylabel('y')
```

#### شرح كل سطر:
1. `import numpy as np` → استيراد مكتبة العمليات الرقمية — نحتاجها لتوليد الأرقام العشوائية والمصفوفات
2. `import matplotlib.pyplot as plt` → استيراد مكتبة الرسم — لعرض البيانات بصريًا
3. `seed = 1` → تثبيت بذرة التوليد العشوائي — لضمان الحصول على نفس البيانات في كل تشغيل
4. `np.random.seed(seed)` → تطبيق البذرة على مولّد الأرقام العشوائية — لتكرارية النتائج
5. `X = np.random.rand(numInstances,1).reshape(-1,1)` → توليد 200 قيمة عشوائية بين 0 و1 — تمثل المتغير المستقل
6. `y_true = -3*X + 1` → حساب القيمة الحقيقية بدون ضجيج — الميل -3 والمقطع 1
7. `y = y_true + np.random.normal(...)` → إضافة ضجيج غاوسي (وسط 0، انحراف معياري 1) — محاكاة عدم الدقة الواقعية
8. `plt.scatter(X, y, color='black')` → رسم النقاط المُشوَّشة (الملاحظات الفعلية)
9. `plt.plot(X, y_true, color='blue', linewidth=3)` → رسم الخط الحقيقي (بدون ضجيج) للمقارنة

**المكتبات المطلوبة (Imports):**
> `numpy`, `matplotlib.pyplot`

**الناتج المتوقع:**
> رسم بياني بنقاط سوداء متناثرة حول خط أزرق سميك ميله سالب (ينحدر من أعلى اليسار إلى أسفل اليمين)، يمثل العلاقة الحقيقية `y = -3x + 1`.

#### شرح كل سطر:
1. `y_pred_test = regr.predict(X_test)` → استخدام النموذج المدرَّب للتنبؤ بقيم `y` لبيانات الاختبار التي لم يرها النموذج
2. `plt.scatter(y_test, y_pred_test, ...)` → رسم القيم الحقيقية مقابل المتوقعة — كلما اقتربت النقاط من خط قطري وهمي (y=x) كان النموذج أدق
3. `np.sqrt(mean_squared_error(y_test, y_pred_test))` → حساب الجذر التربيعي لمتوسط مربع الخطأ (`RMSE`) بين القيم الحقيقية والمتوقعة
4. `r2_score(y_test, y_pred_test)` → حساب معامل التحديد `R²` الذي يقيس نسبة التباين المُفسَّر بواسطة النموذج

**الناتج المتوقع:**
> `Root mean squared error = 1.0476` و`R-squared = 0.4443` — تشير هاتان القيمتان إلى أن النموذج يفسّر حوالي 44% من التباين في بيانات الاختبار، وهو منطقي لأن الضجيج (انحراف معياري 1) كبير نسبيًا مقارنة بحجم الميل (-3) على مدى `x` بين 0 و1.

📐 **المعادلة: الجذر التربيعي لمتوسط مربع الخطأ RMSE**

$$
\text{RMSE} = \sqrt{\frac{1}{N}\sum_{i=1}^{N}\big(y_i - \hat{y}_i\big)^2}
$$

**الشرح:**
> `yᵢ` القيمة الحقيقية | `ŷᵢ` القيمة المتوقعة | يُؤخذ الجذر التربيعي ليكون المقياس بنفس وحدة `y` الأصلية (مثلاً بالدولار وليس بالدولار تربيع)

📐 **المعادلة: معامل التحديد R²**

$$
R^2 = 1 - \frac{\sum_i (y_i - \hat{y}_i)^2}{\sum_i (y_i - \bar{y})^2}
$$

**الشرح:**
> `ȳ` متوسط القيم الحقيقية | البسط هو مجموع مربعات أخطاء النموذج (`RSS`) | المقام هو مجموع مربعات الأخطاء لو استخدمنا المتوسط فقط كتنبؤ ثابت (`TSS`) | كلما اقترب `R²` من 1 كان النموذج أفضل، وكلما اقترب من 0 (أو كان سالبًا) كان أسوأ من مجرد التنبؤ بالمتوسط

#### 💻 الكود: الخطوة 5 — التصور البصري النهائي (Postprocessing)

```python
# Plot outputs

plt.plot(X_test, y_pred_test, color='blue', linewidth=3)

plt.xlabel('X')
plt.ylabel('y')
```

1. `regr.coef_[0][0]` → استخراج الميل المُقدَّر (`ŵ₁`) من النموذج المدرَّب
2. `regr.intercept_[0]` → استخراج المقطع المُقدَّر (`ŵ₀`)
3. `plt.scatter(X_test, y_test, ...)` → رسم نقاط بيانات الاختبار الفعلية
4. `plt.plot(X_test, y_pred_test, ...)` → رسم خط الانحدار المتوقَّع فوق نقاط الاختبار
5. `titlestr = ...` → بناء عنوان يعرض المعادلة المُقدَّرة بالأرقام الفعلية

**الناتج المتوقع:**
> `Slope = -3.2423...` و`Intercept = 1.0806...` — قريبتان جدًا من القيم الحقيقية (`-3` و`1`) رغم التدريب على 20 نقطة فقط، مما يُظهر قدرة الانحدار الخطي على استرجاع العلاقة الحقيقية من بيانات مشوَّشة.

#### 🔍 تتبع التنفيذ: مقارنة القيم الحقيقية بالمُقدَّرة

| المقارنة | Training Error | Test Error |
| --- | --- | --- |
| مصدر البيانات | نفس البيانات المستخدَمة في التدريب | بيانات جديدة لم تُستخدَم في التدريب |
| السلوك مع زيادة تعقيد النموذج | يتناقص دائمًا | يتناقص ثم يرتفع (شكل U) |
| هل يصلح لاختيار النموذج؟ | لا (متفائل زائف) | نعم (تقدير أقرب للواقع) |

### قائمة المصطلحات (Glossary)

| المصطلح الإنجليزي | الترجمة/المعنى |
| --- | --- |
| `Feature Extraction` | استخلاص الميزات من البيانات الخام قبل تغذيتها للنموذج |
| `Hyperplane` | مستوٍ مسطح متعدد الأبعاد يمثل نموذج الانحدار |
| `Loss Function` | دالة الخسارة/التكلفة لخطأ تنبؤ واحد |
| `RSS (Residual Sum of Squares)` | مجموع مربعات البواقي، ما يُصغِّره الانحدار الخطي أثناء التدريب |
| `Generalization Error` | الخطأ الحقيقي المتوقَّع على أي بيانات جديدة (وليس فقط بيانات الاختبار المحدودة) |
| `Overfitting` / `Underfitting` | الإفراط في التخصيص / القصور في التخصيص |
| `Model Complexity` | مدى مرونة/تعقيد النموذج (عدد الميزات، رتبة كثير الحدود) |

### نقاط ذهبية

1. الانحدار يبقى "خطيًا" طالما هو خطي **في الأوزان** `w`، حتى لو كانت الميزات نفسها غير خطية (`x²`, `sin(x)`).
2. خطأ التدريب **لا يصلح أبدًا وحده** لاختيار أفضل نموذج أو تقدير الأداء الحقيقي.
3. تعريف `Overfitting` الدقيق يقارن بين نموذجين: أحدهما أفضل على التدريب لكن أسوأ حقيقيًا.
4. مقولة جورج بوكس: كل النماذج خاطئة، لكن السؤال العملي هو مدى فائدتها رغم خطئها.
5. `R²` و`RMSE` مقياسان مكمّلان: الأول نسبي (0 إلى 1 تقريبًا)، والثاني بنفس وحدة `y`.

### أخطاء شائعة

| الخطأ | لماذا هو خطأ | الصواب |
| --- | --- | --- |
| تقييم النموذج على بيانات التدريب فقط | يعطي انطباعًا متفائلًا زائفًا | استخدم بيانات اختبار منفصلة |
| اعتبار خط مستقيم دائمًا هو "الانحدار الخطي" الوحيد | الانحدار الخطي يشمل أي دالة خطية في الأوزان مهما كانت الميزات غير خطية | تحقق من الميزات `hⱼ(x)` وليس فقط شكل المنحنى |
| زيادة تعقيد النموذج دائمًا لتحسين الدقة | يؤدي إلى `Overfitting` بعد نقطة معينة | راقب خطأ الاختبار وليس خطأ التدريب فقط |

### خطوات وإجراءات المحاضرة

#### ⚙️ خوارزمية: بناء وتقييم نموذج انحدار خطي (عام)

```algorithm
1 | تعريف الميزات | اختيار hⱼ(x) | تحويل x الخام إلى ميزات (ثابت، x، x²، تفاعلات...)
2 | تقسيم البيانات | train/test split | فصل بيانات التدريب عن الاختبار
3 | تدريب النموذج | تصغير RSS على بيانات التدريب | حساب ŵ
4 | حساب خطأ التدريب | L(y_train, f_ŵ(x_train)) | مؤشر متفائل، لا يُعتمَد عليه وحده
5 | حساب خطأ الاختبار | L(y_test, f_ŵ(x_test)) | تقدير أقرب للخطأ الحقيقي
6 | المقارنة عبر مستويات تعقيد مختلفة | رسم Error vs. Model Complexity | اختيار نقطة التوازن (Goldilocks zone)
```

---

### أنماط الأكواد

- **نمط تقسيم البيانات يدويًا:** `X[:-numTest]` / `X[-numTest:]` — مفيد لفهم آلية التقسيم، لكن في التطبيق العملي يُفضَّل `train_test_split` مع `shuffle=True`.
- **نمط "أنشئ ثم درّب ثم تنبأ ثم قيّم"** المتكرر في `scikit-learn`: `Model() → .fit() → .predict() → metric(y_true, y_pred)` — هذا النمط ثابت لكل نماذج `sklearn` تقريبًا (انحدار، تصنيف، تجميع).

### أنماط التعامل

- عند مواجهة بيانات بمدخل واحد لكن العلاقة غير خطية بصريًا (منحنى)، فكّر أولًا في إضافة ميزات (`x²`, `log(x)`,…) قبل تغيير الخوارزمية بالكامل.
- عند مواجهة عدة مدخلات مرتبطة منطقيًا بالمُخرَج (مثل مساحة + حمامات + غرف)، أضفها جميعًا كميزات مستقلة أولًا، ثم افحص التفاعلات إن لزم.

### الأفكار الشاملة

> الفكرة المركزية عبر هذه المحاضرة: **زيادة مرونة النموذج (ميزات أكثر، مدخلات أكثر) تحسّن الملاءمة الظاهرية دائمًا، لكن التقييم الحقيقي للنموذج يجب أن يتم دائمًا على بيانات لم يرها أثناء التدريب.** هذه الفكرة هي الأساس الذي سيُبنى عليه لاحقًا مفهوم `Regularization` (Ridge/Lasso) في محاضرات قادمة، حيث نضيف عقوبة على تعقيد الأوزان لمنع الإفراط في التخصيص دون التخلي عن مرونة النموذج.

**سؤال 16 (سيناريو كود):** ماذا يحدث إذا استخدمنا `regr.predict(X_train)` بدلاً من `regr.predict(X_test)` لتقييم النموذج؟

ب) سنحصل على تقدير أقرب للخطأ الحقيقي
ج) سنحصل على تقييم متفائل زائف لأننا نستخدم بيانات شارك بها النموذج في التدريب
د) سيحدث خطأ برمجي فورًا

**الإجابة الصحيحة: ج**
- أ) خطأ — النتيجة ستختلف لأن `X_train` هي البيانات التي فعليًا تدرَّب عليها النموذج.
- ب) خطأ — العكس تمامًا؛ استخدام بيانات التدريب يعطي تقييمًا متفائلًا وليس واقعيًا.
- ج) صحيح — هذا هو بالضبط تعريف `Training Error` المتفائل الذي نُحذّر منه في البند 5.
- د) خطأ — لن يحدث خطأ برمجي، فقط النتيجة ستكون مضلِّلة إحصائيًا.

**تتبّع خطوة بخطوة (أكمل الجدول):**
| الخطوة | العملية | الحالة |
| --- | --- | --- |

**نموذج الحل:**
| الخطوة | العملية | الحالة |

**النتيجة:** معادلة النموذج النهائية جاهزة للاستخدام في التنبؤ ورسمها بيانيًا.

## الجزء الخامس: أسئلة نظرية متوقعة بالامتحان

### السؤال 1: ما المقصود بـ `Feature` في سياق الانحدار، ولماذا لا تقتصر على `x` نفسه؟
**نموذج الإجابة:** 1. التعريف: دالة `hⱼ(x)` تُشتَقّ من المدخل الأصلي وتُستخدم كمُدخل فعلي للنموذج. 2. المكونات: يمكن أن تكون ثابتًا، `x` نفسه، `x²`، `sin(x)`، أو تفاعلًا بين متغيرين. 3. مثال رقمي: `h₀(x)=1`, `h₁(x)=x`, `h₂(x)=x²`. 4. متى نستخدمها: عندما تكون العلاقة الحقيقية بين المدخل والمخرج غير خطية بشكل بسيط.

### السؤال 2: عرّف `Hyperplane` ووضّح متى يظهر في نموذج الانحدار.
**نموذج الإجابة:** 1. التعريف: سطح مستوٍ (غير منحنٍ) يمثل نموذج الانحدار في فضاء متعدد الأبعاد. 2. المكونات: يتحدد بالأوزان `w₀, w₁, ..., w_d` ومدخلات `x[1],...,x[d]`. 3. مثال رقمي: بمدخلين (مساحة وحمامات) يصبح مستوى ثلاثي الأبعاد. 4. متى نستخدمه: عند وجود أكثر من مدخل واحد في نموذج انحدار خطي.

### السؤال 3: ما الفرق بين `Loss Function` و`Training Error`؟
**نموذج الإجابة:** 1. التعريف: `Loss` تقيس الخطأ عند نقطة بيانات واحدة، بينما `Training Error` هو متوسط الخسارة عبر كل نقاط التدريب. 2. المكونات: `L(y,f_ŵ(x))` لكل نقطة مقابل `(1/N)Σ L(...)` للجميع. 3. مثال رقمي: خسارة نقطة واحدة = `(y-ŷ)²`؛ خطأ التدريب = متوسط هذه القيمة على N نقطة. 4. متى نستخدم كل منهما: `Loss` لتعريف المشكلة رياضيًا، و`Training Error` لتلخيص الأداء الكلي على مجموعة بيانات.

### السؤال 4: لماذا يُعتبر خطأ التدريب "متفائلًا بشكل زائف" (`overly optimistic`)؟
**نموذج الإجابة:** 1. التعريف: لأن الأوزان `ŵ` نفسها اختيرت لتُصغّر هذا الخطأ تحديدًا. 2. المكونات: عملية `.fit()` تُصغّر `RSS` على بيانات التدريب فقط. 3. مثال رقمي: زيادة رتبة كثير الحدود تُخفّض `Training Error` باستمرار حتى قرب الصفر. 4. متى يظهر ذلك بوضوح: عند مقارنة نماذج بدرجات تعقيد مختلفة دون قياس خطأ الاختبار.

### السؤال 5: عرّف `Test Error` ووضّح علاقته بـ `Generalization Error`.
**نموذج الإجابة:** 1. التعريف: متوسط الخسارة على بيانات لم يرها النموذج أثناء التدريب. 2. المكونات: يُحسَب بنفس دالة الخسارة (غالبًا مربع الخطأ) لكن على `X_test, y_test`. 3. مثال رقمي: `Test error = (1/N)Σ(y_test,i - f_ŵ(x_test,i))²`. 4. متى نستخدمه: كتقدير عملي وأقرب للخطأ الحقيقي (`Generalization Error`) الذي لا يمكن حسابه فعليًا لأنه يتطلب كل البيانات الممكنة في العالم.

### السؤال 6: اشرح التعريف الرسمي لـ `Overfitting` بالكامل.
**نموذج الإجابة:** 1. التعريف: يحدث الإفراط في التخصيص عند وجود نموذج بديل `w'` يحقق أداءً أفضل حقيقيًا رغم أدائه الأسوأ على التدريب مقارنة بـ `ŵ`. 2. الشرطان: `(1) training_error(ŵ) < training_error(w')` و`(2) true_error(ŵ) > true_error(w')`. 3. مثال رقمي: `ŵ` (رتبة 15): تدريب=0.01، حقيقي=5.0 | `w'` (رتبة 2): تدريب=0.5، حقيقي=1.0 → `ŵ` مُفرط في التخصيص. 4. متى نستخدم هذا التعريف: لاختيار نموذج أنسب من بين عدة نماذج مرشحة بموضوعية.

### السؤال 7: قارن بين `Underfitting` و`Overfitting` من حيث السبب والعلاج.
**نموذج الإجابة:** 1. التعريف: `Underfitting` = نموذج بسيط جدًا لا يلتقط النمط؛ `Overfitting` = نموذج معقد جدًا يلتقط الضجيج. 2. المكونات: الأول خطأ تدريب واختبار مرتفعان معًا؛ الثاني خطأ تدريب منخفض جدًا لكن اختبار مرتفع. 3. مثال رقمي: خط مستقيم لبيانات منحنية (Underfitting) مقابل كثير حدود رتبة 20 (Overfitting). 4. العلاج: زيادة التعقيد/الميزات للأول، وتقليل التعقيد أو استخدام `Regularization`/زيادة البيانات للثاني.

### السؤال 8: ما دور `scikit-learn` في سير عمل الانحدار، واذكر أهم دالتين تُستخدَمان.
**نموذج الإجابة:** 1. التعريف: مكتبة بايثون توفّر أدوات جاهزة لبناء وتقييم نماذج التعلم الآلي. 2. المكونات: `linear_model.LinearRegression()` لإنشاء النموذج، و`.fit()`/`.predict()` للتدريب والتنبؤ، و`mean_squared_error`/`r2_score` للتقييم. 3. مثال رقمي: `regr.fit(X_train,y_train)` ثم `regr.predict(X_test)`. 4. متى نستخدمها: في كل مشروع انحدار عملي بدل كتابة معادلات التصغير يدويًا.

### السؤال 9: ما الفرق بين `RMSE` و`R²` من حيث التفسير؟
**نموذج الإجابة:** 1. التعريف: `RMSE` مقياس مطلق بنفس وحدة `y`؛ `R²` مقياس نسبي بين 0 و1 تقريبًا. 2. المكونات: `RMSE=√MSE`؛ `R² = 1 - RSS/TSS`. 3. مثال رقمي: `RMSE=1.0476` (بوحدة `y` نفسها) مقابل `R²=0.4443` (44% من التباين مُفسَّر). 4. متى نستخدم كلًا منهما: `RMSE` لفهم حجم الخطأ الفعلي بوحدات مفهومة، و`R²` لمقارنة نسبية بين نماذج مختلفة بغض النظر عن وحدة القياس.

### السؤال 10: لماذا نستخدم بيانات اصطناعية (`Synthetic Data`) لتعليم الانحدار الخطي بدل بيانات حقيقية مباشرة؟
**نموذج الإجابة:** 1. التعريف: بيانات مولَّدة رياضيًا بمعادلة معروفة مسبقًا (`y=-3x+1+ε`). 2. المكونات: متغير مستقل عشوائي `x`، دالة حقيقية معروفة، ضجيج غاوسي `ε`. 3. مثال رقمي: بعد التدريب حصلنا على `slope≈-3.24` قريب من `-3` الحقيقي. 4. متى نستخدمها: لأغراض تعليمية وتحقق من صحة الخوارزمية، لأننا نعرف "الإجابة الصحيحة" مسبقًا ويمكننا قياس دقة الاسترجاع مباشرة.

## بطاقات سؤال وجواب (Q&A Cards)

**Q1:** ما الفرق بين `Classification` و`Regression`؟
A: `Regression` يتنبأ بقيمة مستمرة (مثل السعر)، بينما `Classification` يتنبأ بفئة مُنفصلة (مثل نعم/لا).

**Q2:** ما هو `Feature` في سياق الانحدار؟
A: دالة `hⱼ(x)` مُشتقّة من المدخل الأصلي تُستخدم كمُدخل فعلي للنموذج، قد تكون `x` نفسه أو `x²` أو غيرهما.

**Q3:** ما الفرق بين `Training Error` و`Test Error`؟
A: `Training Error` يُقاس على بيانات التدريب (متفائل)، و`Test Error` يُقاس على بيانات جديدة (يقارب الواقع).

**Q4:** لماذا لا نثق بخطأ التدريب وحده؟
A: لأن الأوزان `ŵ` اختيرت خصيصًا لتصغيره، فينخفض باستمرار مع زيادة التعقيد بغض النظر عن الجودة الحقيقية.

**Q5:** ما تعريف `Overfitting` الرسمي؟
A: وجود نموذج بديل `w'` أداؤه أسوأ على التدريب لكن أفضل على الخطأ الحقيقي مقارنة بالنموذج الحالي `ŵ`.

**Q6:** ما شكل منحنى Test/True Error مقابل تعقيد النموذج؟
A: على شكل حرف U — يتناقص أولًا ثم يرتفع مجددًا بعد نقطة معينة.

**Q7:** ما الفرق بين `RMSE` و`MSE`؟
A: `RMSE` هو الجذر التربيعي لـ `MSE`، مما يجعله بنفس وحدة `y` الأصلية.

**Q8:** ما معنى `R² = 0.44`؟
A: يعني أن النموذج يفسّر حوالي 44% من التباين الموجود في بيانات الاختبار.

**Q9:** ما هو `Hyperplane`؟
A: السطح المستوي الذي يمثل نموذج الانحدار في فضاء متعدد الأبعاد عند وجود أكثر من مدخل واحد.

**Q10:** ما وظيفة `.fit()` في `scikit-learn`؟
A: تدريب النموذج عبر تصغير `RSS` (أو دالة خسارة أخرى) على بيانات التدريب المُعطاة له.

**Q11:** ما وظيفة `.predict()`؟
A: تطبيق النموذج المدرَّب على بيانات جديدة (عادة `X_test`) لإنتاج تنبؤات `ŷ`.

**Q12:** لماذا نضيف ضجيجًا (`epsilon`) عند توليد بيانات اصطناعية؟
A: لمحاكاة عدم الدقة/التشويش الموجود طبيعيًا في البيانات الحقيقية.

**Q13:** ما الفرق بين `Underfitting` و`Overfitting` من حيث خطأ التدريب؟
A: `Underfitting` يعطي خطأ تدريب مرتفعًا؛ `Overfitting` يعطي خطأ تدريب منخفضًا جدًا (لكن خطأ اختبار مرتفعًا).

**Q14:** ماذا تعني مقولة جورج بوكس "All models are wrong"؟
A: أن كل نموذج تقريب غير مثالي، والسؤال العملي هو مدى فائدته رغم عدم كماله، وليس السعي لنموذج مثالي غير موجود.

**Q15:** ما الفرق بين `x[j]` و`hⱼ(x)`؟
A: `x[j]` هو المدخل الخام رقم j، بينما `hⱼ(x)` هي الميزة المُشتقّة (دالة) من المدخلات — قد تكون مطابقة لـ `x[j]` أو أكثر تعقيدًا.

**Q16:** ما الهدف من مرحلة `Postprocessing` في خطوات scikit-learn؟
A: تصوّر النموذج المُدرَّب بصريًا (رسم الخط/المنحنى النهائي فوق بيانات الاختبار) لفهم جودته بشكل بديهي.

## كتابة الكود الكامل (مرجع شامل)

> الكود الكامل مجمّع من جميع أجزاء المحاضرة — مرجع للطالب، لا شرح جديد.

```python
# === Synthetic Data Generation ===
%matplotlib inline
import numpy as np
import matplotlib.pyplot as plt

seed = 1
numInstances = 200
np.random.seed(seed)
X = np.random.rand(numInstances, 1).reshape(-1, 1)
y_true = -3 * X + 1
y = y_true + np.random.normal(size=numInstances).reshape(-1, 1)

plt.scatter(X, y, color='black')
plt.plot(X, y_true, color='blue', linewidth=3)
plt.title('True function: y = -3X + 1')
plt.xlabel('X')
plt.ylabel('y')

# === Regression: Train/Test Split ===
numTrain = 20
numTest = numInstances - numTrain

X_train = X[:-numTest]
X_test = X[-numTest:]
y_train = y[:-numTest]
y_test = y[-numTest:]

# === Regression: Fit Model ===
from sklearn import linear_model
from sklearn.metrics import mean_squared_error, r2_score

regr = linear_model.LinearRegression()
regr.fit(X_train, y_train)

# === Regression: Apply and Evaluate ===
y_pred_test = regr.predict(X_test)

plt.scatter(y_test, y_pred_test, color='black')
plt.title('Comparing true and predicted values for test set')
plt.xlabel('True values for y')
plt.ylabel('Predicted values for y')

print("Root mean squared error = %.4f" % np.sqrt(mean_squared_error(y_test, y_pred_test)))
print('R-squared = %.4f' % r2_score(y_test, y_pred_test))

# === Regression: Postprocessing (Visualize Fitted Model) ===
print('Slope = ', regr.coef_[0][0])
print('Intercept = ', regr.intercept_[0])

plt.scatter(X_test, y_test, color='black')
plt.plot(X_test, y_pred_test, color='blue', linewidth=3)
titlestr = 'Predicted Function: y = %.2fX + %.2f' % (regr.coef_[0], regr.intercept_[0])
plt.title(titlestr)
plt.xlabel('X')
plt.ylabel('y')
```
