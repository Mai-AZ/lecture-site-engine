### 1. مشكلة اختيار أفضل خط (Fitting a Line to Data)

#### النص الأصلي يقول:
> "For a given training dataset, there will be multiple lines and each one is given by a different set of parameters 𝑊̂. which is the best line to use? Or which 𝑊̂ do we have to choose for our model?"

#### الشرح المبسّط:
عندنا بيانات (مثلاً مساحة البيت بالـ `sq.ft.` وسعره بالدولار)، ولو رسمنا خطوط عديدة تحاول كلها "تمر" وسط النقاط، كل خط له معادلته الخاصة بشكل `y = w0 + w1*x`. السؤال: أي خط من هذه الخطوط هو الأفضل لتمثيل العلاقة بين المساحة والسعر؟

**لماذا؟** لأن كل قيمتين مختلفتين لـ `w0` (نقطة التقاطع مع المحور y) و `w1` (الميل/Slope) تنتجان خطاً مختلفاً تماماً، وبدون معيار واضح للمقارنة، لا نستطيع أن نختار بينها بشكل موضوعي.

#### 💡 التشبيه:
> تخيل أنك تحاول أن تصمم خط سباق (مضمار) يمر وسط مجموعة أعمدة موزعة بشكل عشوائي تقريباً على خط مستقيم؛ ممكن ترسم عشرات المضامير المختلفة، لكن تحتاج "حكم" يقيس أي مضمار أقرب للأعمدة ككل.
> **وجه الشبه:** الأعمدة = نقاط البيانات، المضمار = الخط `y = w0 + w1*x`.

🤔 **تفعيل الفهم (اسأل نفسك):**
> **سؤال:** إذا كان عندي خطين مختلفين يمران قريبين من نفس البيانات، كيف أقرر أيهما "أفضل" رياضياً وليس بالعين المجردة؟
> **لماذا هذا مهم؟** لأن اختيار النموذج يدوياً بالعين غير قابل للتكرار أو الأتمتة، والحاسوب يحتاج معياراً رقمياً.

### 2. مقياس الجودة (Quality Metric) و مجموع مربعات البواقي (RSS)

#### النص الأصلي يقول:
> "we need a quality metric to measure the cost associated with a specific line (fit), e.g., residual sum of squares (RSS). The residual is the difference between the prediction value and actual value (error). RSS Simply adds up the square errors over all the different training data points."

#### الشرح المبسّط:
لحل مشكلة "أي خط أفضل"، نحتاج **رقماً واحداً** يلخّص مدى سوء (أو جودة) الخط. هذا الرقم اسمه `RSS` (Residual Sum of Squares). فكرته:
1. لكل نقطة بيانات (بيت)، نحسب `residual` = الفرق بين السعر الحقيقي `y` والسعر المتوقع من الخط `ŷ = w0 + w1*x`.
2. نربّع هذا الفرق (حتى لا تلغي الأخطاء الموجبة السالبة بعضها، ولحتى نعاقب الأخطاء الكبيرة أكثر).
3. نجمع كل هذه المربعات على كل نقاط التدريب.

**لماذا نربّع الخطأ بدل أخذ القيمة المطلقة؟** لأن التربيع يجعل الدالة قابلة للاشتقاق بسهولة (smooth)، وهذا ضروري لاحقاً عندما نستخدم المشتقات لإيجاد أفضل حل (`Gradient Descent` أو `closed-form`).

#### 📐 المعادلة: Residual Sum of Squares (RSS)

$$
RSS(w_0, w_1) = \sum_{i=1}^{N} \left(y_i - [w_0 + w_1 x_i]\right)^2
$$

**الشرح:**
> - `y_i`: السعر الحقيقي للبيت رقم `i` (Actual value).
> - `w_0 + w_1*x_i`: السعر المتوقع من الخط (Prediction) عند مساحة `x_i`.
> - `(y_i - [w_0+w_1*x_i])`: هو الـ `residual` (الخطأ) لهذه النقطة.
> - `N`: عدد كل نقاط بيانات التدريب.
> - الجمع `Σ` من `i=1` إلى `N`: يجمع مربعات كل الأخطاء ليعطي رقماً واحداً يمثل "سوء" الخط بالكامل.

#### 💡 التشبيه:
> تخيل أن `RSS` مثل "فاتورة غرامة" على كل خط تجربه؛ كل نقطة بعيدة عن الخط تدفع غرامة (والبعد الكبير غرامته أكبر بكثير بسبب التربيع، مثل السرعة الزائدة في المخالفات المرورية كلما زادت السرعة زادت الغرامة بشكل غير خطي).
> **وجه الشبه:** الغرامة الكلية = `RSS`، والمخالفة الفردية = `residual` المربّع لكل بيت.

#### مهم للامتحان ⚠️:
> `RSS` هي دالة تعتمد على **معاملين فقط** هما `w0` و `w1` (وليس على `x` أو `y` بعد أن نثبت البيانات)؛ أي أن `RSS` تتغير فقط عندما نغيّر اختيارنا للخط، وليس عندما تتغير بيانات التدريب نفسها.

### 3. صياغة المشكلة كمسألة تحسين (Optimization Problem)

#### النص الأصلي يقول:
> "search over all possible lines, all w0 and w1, and trying to find the one that results in the smallest RSS." ... "Over the entire space, we want to find the point (Ŵ0, Ŵ1) that minimize the RSS."

#### الشرح المبسّط:
بما أن `RSS(w0, w1)` هي دالة رياضية بمُتغيّرين (`w0` و `w1`)، فإن السؤال "ما هو أفضل خط؟" أصبح رياضياً: **"ما هي قيم `w0` و `w1` التي تجعل قيمة `RSS` أصغر ما يمكن؟"**. هذا يسمى بمسألة تحسين (Optimization Problem) من نوع Minimization.

**لماذا هذا التحويل مهم؟** لأنه يحوّل مشكلة "غامضة" (اختيار خط جيد) إلى مشكلة رياضية دقيقة لها أدوات حل معروفة (Calculus / Optimization).

#### 📐 المعادلة: صياغة التحسين

$$
\min_{w_0, w_1} \sum_{i=1}^{N} (y_i - [w_0 + w_1 x_i])^2
$$

**الشرح:**
> - `min` تحت `w0, w1`: يعني أننا نبحث بين كل القيم الممكنة لـ `w0` و `w1` عن التوليفة التي تعطي أصغر `RSS`.
> - إذا رسمنا `RSS` كدالة في فضاء ثلاثي الأبعاد (محاور `w0`, `w1`, و `RSS`)، فإنها تأخذ شكل "وعاء" (bowl shape) لأنها `convex function`، وقعر هذا الوعاء هو نقطة الحل المثلى `(Ŵ0, Ŵ1)`.

#### 💡 التشبيه:
> تخيل كرة رخام تتدحرج داخل وعاء (صحن عميق)؛ أياً كانت نقطة انطلاقها، ستستقر في النهاية في أعمق نقطة بالوعاء.
> **وجه الشبه:** الكرة = القيم الحالية لـ `(w0, w1)`، الوعاء = دالة `RSS`، أعمق نقطة = الحل الأمثل `(Ŵ0, Ŵ1)`.

### 4. الأمثلية أحادية البعد (Optimization for One Dimensional Objectives)

#### النص الأصلي يقول:
> "Approach 1 (via derivative): Assuming a convex/concave function, set the derivative of the function equal to zero and the solution will be unique."

#### الشرح المبسّط:
قبل الانتقال إلى دالة بمتغيرين (`w0`, `w1`)، المحاضرة تشرح الفكرة أولاً على دالة بمتغير واحد `g(w)`. الفكرة الرياضية الأساسية: **عند القمة (max) أو القاع (min) لدالة معينة، يكون ميل المماس (Derivative) يساوي صفراً بالضبط.**
- لو الدالة `Convex` (شكل وعاء) → عندها نقطة `min` واحدة، ومشتقتها = صفر عندها فقط.
- لو الدالة `Concave` (شكل قبة) → عندها نقطة `max` واحدة، ومشتقتها = صفر عندها فقط.
- لو الدالة ليست `convex` ولا `concave` (Neither) → ممكن يكون فيه عدة حلول لـ `derivative=0` (Multiple local solutions)، أو ما فيه حل واضح إطلاقاً.

**لماذا هذا الشرط (convex/concave) مهم؟** لأنه يضمن أن الحل الذي نجده بجعل المشتقة = صفر هو **الحل الوحيد والعام (Global)**، وليس مجرد حل محلي (Local) قد يخدعنا.

#### 💡 التشبيه:
> تخيل طريقاً جبلياً بقمة واحدة فقط (Concave) — أعلى نقطة فيه واضحة ولا لبس فيها. أما لو الطريق فيه عدة قمم وأودية متعرجة (Neither)، فقد تظن أنك في القمة بينما هناك قمة أعلى بمكان آخر.
> **وجه الشبه:** القمة الواحدة الواضحة = دالة `Concave`، القمم المتعددة المتعرجة = دالة `Neither` وفيها حلول محلية مضلِّلة.

#### الفهم الخاطئ الشائع ❌:
> "لو حسبت المشتقة وساويتها بصفر ولقيت حل، فهذا يعني تلقائياً أنه الحل الأفضل (Global) بغض النظر عن شكل الدالة."

#### الفهم الصحيح ✅:
> جعل المشتقة تساوي صفراً يعطي حلاً وحيداً وعاماً **فقط** إذا كانت الدالة `convex` (للتصغير) أو `concave` (للتكبير)؛ في الدوال غير المحدبة/المقعرة، قد نحصل على عدة نقاط تحقق `derivative=0` وليست كلها الحل الأفضل.

### 5. خوارزمية التسلق (Hill Climbing) وخوارزمية النزول (Hill Descent)

#### النص الأصلي يقول:
> "Approach 2: Finding the max via hill climbing algorithm... To know whether to move w to the right or left (increase or decrease w), we take the derivative and if the derivative: Positive(left of the optimal) => increase w. Negative(right of the optimal) => decrease w. Zero => exactly at the optimum"

#### الشرح المبسّط:
بدلاً من حل معادلة `derivative=0` مباشرة (وهذا غير ممكن دائماً في مسائل تعلم الآلة المعقدة)، نستخدم خوارزمية **تكرارية (Iterative)** تبدأ من قيمة عشوائية لـ `w` ثم "تتسلق" أو "تنزل" تدريجياً حتى تصل للقمة أو القاع.

القاعدة البسيطة (لإيجاد `max` عبر `Hill Climbing`):
- إذا كانت المشتقة **موجبة** → نحن على يسار القمة → يجب أن **نزيد** `w`.
- إذا كانت المشتقة **سالبة** → نحن على يمين القمة → يجب أن **ننقص** `w`.
- إذا كانت المشتقة **صفر** → وصلنا بالضبط للقمة (Optimum).

**لماذا؟** لأن إشارة المشتقة تخبرنا في أي اتجاه "ترتفع" الدالة؛ فإذا مشينا بنفس اتجاه المشتقة الموجبة، نقترب من القمة أكثر فأكثر.

```algorithm
1 | ابدأ بقيمة ابتدائية عشوائية | تحديد نقطة انطلاق | w = w^(0)
2 | احسب المشتقة عند القيمة الحالية | تفاضل | dg/dw عند w^(t)
3 | حدّث القيمة حسب اتجاه المشتقة | Hill Climbing rule | w^(t+1) = w^(t) + η * dg/dw
4 | كرر الخطوتين 2 و3 حتى التقارب | تكرار | توقف عندما تصبح المشتقة قريبة من صفر
```

#### نقاط التنفيذ:
- خوارزمية `Hill Climbing` تبحث عن `max` (تتحرك في **نفس** اتجاه إشارة المشتقة).
- خوارزمية `Hill Descent` تبحث عن `min` (تتحرك في **عكس** اتجاه إشارة المشتقة تماماً).

#### النص الأصلي يقول:
> "Approach 2: Finding the min via hill descent algorithm... Same as the hill climbing. But, instead moving in the same sign of the derivative, we're going to move in the opposite."

#### الشرح المبسّط:
`Hill Descent` هي نفس فكرة `Hill Climbing` تماماً، لكن معكوسة الاتجاه؛ لأننا هذه المرة نريد الوصول لأخفض نقطة (`min`) وليس أعلى نقطة (`max`).
- المشتقة **موجبة** (يمين الحل الأمثل) → **ننقص** `w`.
- المشتقة **سالبة** (يسار الحل الأمثل) → **نزيد** `w`.
- المشتقة **صفر** → وصلنا للحل الأمثل بالضبط.

```algorithm
1 | ابدأ بقيمة ابتدائية عشوائية | تحديد نقطة انطلاق | w = w^(0)
2 | احسب المشتقة عند القيمة الحالية | تفاضل | dg/dw عند w^(t)
3 | حدّث القيمة بعكس اتجاه المشتقة | Hill Descent rule | w^(t+1) = w^(t) - η * dg/dw
4 | كرر حتى التقارب | تكرار | توقف عندما |dg/dw| < ε
```

#### 📐 المعادلة: قاعدة تحديث Hill Descent

$$
w^{(t+1)} \leftarrow w^{(t)} - \eta \frac{dg}{dw}\Big|_{w^{(t)}}
$$

**الشرح:**
> - `w^(t)`: قيمة `w` عند التكرار (Iteration) رقم `t`.
> - `η` (Step size / Learning rate): رقم صغير موجب يتحكم بحجم الخطوة في كل تحديث.
> - `dg/dw at w^(t)`: المشتقة عند القيمة الحالية، وهي التي تحدد اتجاه ومقدار التعديل.
> - الإشارة **السالبة** قبل `η` هي التي تجعلها `Descent` (نزول للـ `min`) بدل `Climbing`.

#### 💡 التشبيه:
> تخيل نفسك تمشي في الظلام على منحدر، ولا ترى شيئاً إلا ميل الأرض تحت قدميك؛ إذا شعرت أن الأرض تميل نزولاً لجهة معينة، تخطو خطوة بذلك الاتجاه، وتستمر خطوة خطوة حتى تصل لأخفض نقطة ولا تعود تشعر بأي ميل.
> **وجه الشبه:** ميل الأرض تحت قدميك = المشتقة (`derivative`)، طول الخطوة = `step size η`، أخفض نقطة = الحل الأمثل.

### 6. حجم الخطوة ومعيار التقارب (Step Size & Convergence Criteria)

#### النص الأصلي يقول:
> "Fixed step size: equal to some constant (e.g., η=0.1). For a very big step, we may end up jumping over the optimal many times => we converge very slowly to the optimal. Decreasing step size: decrease the step size as the number of iterations increase. be careful about is not decreasing the step size too rapidly => we converge very slowly to the optimal."

#### الشرح المبسّط:
اختيار قيمة `η` (step size) له تأثير كبير على أداء الخوارزمية:
1. **Fixed step size**: نستخدم رقماً ثابتاً طوال التنفيذ (مثلاً `η=0.1`). المشكلة: لو كانت الخطوة كبيرة جداً، قد نقفز فوق الحل الأمثل ذهاباً وإياباً (Overshooting) بدل الاستقرار عليه، فنتأخر كثيراً في التقارب.
2. **Decreasing step size**: نجعل `η` يصغر تدريجياً كلما زاد عدد التكرارات (مثلاً `η_t = α/t` أو `η_t = α/√t`). لكن يجب الحذر: إذا صغّرنا `η` بسرعة كبيرة جداً منذ البداية، فإن الخطوات تصبح صغيرة جداً فتتباطأ الخوارزمية كثيراً قبل الوصول للحل.

**لماذا هذه المقايضة موجودة؟** لأن خطوة كبيرة تُسرّع الاقتراب في البداية لكنها تمنع الاستقرار الدقيق بالنهاية، بينما خطوة صغيرة جداً تعطي دقة لكنها بطيئة جداً؛ لذلك عادة نبدأ بخطوة أكبر ثم نصغّرها تدريجياً.

#### ⚖️ المقايضة: Fixed Step Size مقابل Decreasing Step Size

| | Fixed Step Size | Decreasing Step Size |
| --- | --- | --- |
| المزايا | بسيطة التطبيق، لا تحتاج جدولة معقدة | تقارب أدق قرب الحل الأمثل، تتجنب القفز الزائد |
| العيوب | قد تسبب Overshooting وتذبذب حول الحل | إذا نقصت بسرعة كبيرة، التقارب يصبح بطيئاً جداً |
| متى تختاره | عندما تكون الدالة بسيطة وواضحة الشكل | عندما نريد دقة عالية في الحل النهائي |

#### 📐 المعادلة: معيار التقارب (Convergence Criteria)

$$
\text{Optimum occurs when } \frac{dg(w)}{dw} = 0 \quad \text{, in practice stop when } \left|\frac{dg(w)}{dw}\right| < \epsilon
$$

**الشرح:**
> - نظرياً، الحل الأمثل عندما تكون المشتقة تساوي صفراً بالضبط.
> - عملياً (Practically)، لا نصل أبداً لصفر تام بسبب التقريب العددي (numerical computation)، لذلك نوقف الخوارزمية عندما تصبح القيمة المطلقة للمشتقة أصغر من عتبة صغيرة جداً (threshold) نرمز لها بـ `ε` (epsilon)، ونحددها مسبقاً حسب دقة المطلوبة.

#### مهم للامتحان ⚠️:
> `ε` ليست قيمة ثابتة عالمياً، بل هي **عتبة (threshold) يحددها المصمم** حسب حاجته للدقة؛ كلما كانت أصغر، كانت الدقة أعلى لكن عدد التكرارات (وبالتالي الوقت) أكبر.

### 7. الانتقال إلى أبعاد متعددة (Moving to Multiple Dimensions) وخوارزمية Gradient Descent

#### النص الأصلي يقول:
> "Gradient descent algorithm: Analogous to the hill decent algorithm in 1D. Everything exactly the same, but instead of the derivative, we've the gradient. Convergence criteria: if the magnitude of the gradient is less than some ε, then algorithm will be converged."

#### الشرح المبسّط:
في مسألتنا، `RSS` ليست دالة بمتغير واحد بل بمتغيرين (`w0` و `w1`)؛ لذلك بدل المشتقة العادية (Derivative) نستخدم مفهوماً أعمّ اسمه **التدرج (Gradient)**، وهو ببساطة **مُتّجه (vector)** يحتوي كل المشتقات الجزئية (Partial Derivatives) بالنسبة لكل معامل على حدة.

**ما هي المشتقة الجزئية (Partial Derivative)؟** هي مثل المشتقة العادية تماماً، لكن نفترض أن كل المتغيرات الأخرى ثابتة (constant) ونشتق فقط بالنسبة لمتغير واحد في كل مرة.

**لماذا نحتاج التدرج بدل مشتقة واحدة؟** لأنه في أكثر من بُعد، لا يوجد "اتجاه واحد" فقط (يمين/يسار)؛ التدرج يخبرنا الاتجاه في الفضاء متعدد الأبعاد الذي تزداد فيه الدالة بأسرع ما يمكن (وعكسه هو اتجاه أسرع تناقص).

```algorithm
1 | ابدأ بقيم ابتدائية للمعاملات | تحديد نقطة انطلاق | w = w^(0) = [w0^(0), w1^(0)]
2 | احسب التدرج عند القيم الحالية | تفاضل جزئي لكل معامل | ∇g(w^(t))
3 | حدّث كل المعاملات معاً بعكس اتجاه التدرج | Gradient Descent rule | w^(t+1) = w^(t) - η ∇g(w^(t))
4 | كرر حتى تصغر قيمة التدرج | فحص التقارب | توقف عندما ||∇g(w)|| < ε
```

#### 📐 المعادلة: التدرج (Gradient) وقاعدة Gradient Descent

$$
\nabla g(w) = \begin{bmatrix} \partial g/\partial w_0 \\ \partial g/\partial w_1 \\ \vdots \\ \partial g/\partial w_p \end{bmatrix}
\quad , \quad
w^{(t+1)} \leftarrow w^{(t)} - \eta \nabla g(w^{(t)})
$$

**الشرح:**
> - `∇g(w)`: متجه التدرج (Gradient vector)، يحتوي مشتقة جزئية واحدة لكل معامل من `w0` إلى `wp` (إذا كان عندنا `p+1` معامل، فمتّجه التدرج بُعده `(p+1)`).
> - `∂g/∂w0`: المشتقة الجزئية بالنسبة لـ `w0` فقط، مع اعتبار باقي المعاملات (مثل `w1`) ثوابت.
> - قاعدة التحديث مطابقة تماماً لقاعدة `Hill Descent`، فقط استبدلنا "المشتقة العادية" بـ "متجه التدرج"، لأننا الآن نحدّث **كل المعاملات دفعة واحدة**.
> - **معيار التقارب**: نتحقق من **مقدار (magnitude/norm)** متجه التدرج `||∇g(w)||`؛ إذا أصبح أصغر من `ε`، توقفنا لأننا أصبحنا قريبين جداً من نقطة التدرج=صفر (الحل الأمثل).

#### 💡 التشبيه:
> تخيل أنك تمشي في الظلام لكن هذه المرة على تضاريس ثلاثية الأبعاد وليس خطاً واحداً (مثل جبل حقيقي)؛ في كل نقطة تشعر بأكثر من اتجاه للميل (شمال-جنوب وشرق-غرب معاً)، والتدرج يخبرك الاتجاه المُركّب الذي يعطيك أسرع صعود، وأنت تمشي بعكسه للنزول بأسرع طريق.
> **وجه الشبه:** التضاريس ثلاثية الأبعاد = دالة `RSS(w0,w1)`، الاتجاه المُركّب للميل = `Gradient vector`.

### 8. تطبيق Gradient Descent على مسألة RSS الفعلية

#### النص الأصلي يقول:
> "An optimization (minimization) problem: a convex function in multidimensional space. solution is unique. Approach 1: Set gradient = 0. Approach 2: Gradient descent algorithm"

#### الشرح المبسّط:
لأن `RSS(w0,w1)` هي دالة `convex` (شكل وعاء) حتى في الأبعاد المتعددة، فإن **الحل وحيد (unique)** ولا يوجد قلق من الوقوع في حل محلي خاطئ. عندنا طريقتان لإيجاد هذا الحل الوحيد:
- **Approach 1**: نساوي التدرج بصفر ونحل جبرياً مباشرة (Closed-form solution).
- **Approach 2**: نستخدم `Gradient Descent` بشكل تكراري.

#### النص الأصلي يقول:
> "Compute the gradient: ∇RSS(w0,w1) = [-2 Σ[yi - (w0+w1xi)], -2 Σ[yi - (w0+w1xi)]xi]"

#### الشرح المبسّط:
نحسب المشتقة الجزئية لدالة `RSS` بالنسبة لكل من `w0` و `w1` باستخدام قاعدة السلسلة (chain rule) على كل حد داخل الجمع (ونستخدم حقيقة أن مشتقة الجمع = جمع المشتقات).

#### 📐 المعادلة: تدرج RSS

$$
\nabla RSS(w_0, w_1) = \begin{bmatrix} -2 \sum_{i=1}^{N} [y_i - (w_0+w_1 x_i)] \\ -2 \sum_{i=1}^{N} [y_i - (w_0+w_1 x_i)] x_i \end{bmatrix}
$$

**الشرح:**
> - العنصر الأول (بالنسبة لـ `w0`): مجموع الأخطاء (residuals) كلها مضروبة بـ `-2`.
> - العنصر الثاني (بالنسبة لـ `w1`): مجموع الأخطاء لكن كل خطأ مضروب أولاً بـ `xi` المقابل له (بسبب قاعدة السلسلة حين نشتق بالنسبة لـ `w1` التي تُضرب بـ `xi` في المعادلة الأصلية) ثم كل ذلك مضروب بـ `-2`.

#### النص الأصلي يقول:
> "Approach 1: Set gradient = 0 and Solve for W0 and W1 => Those are going to be our estimated parameters for our selected model."

#### الشرح المبسّط:
نساوي كل عنصر من متجه التدرج بصفر (نظام من معادلتين بمجهولين)، ونحل جبرياً لنحصل على صيغة مباشرة (Closed-form) لـ `Ŵ0` و `Ŵ1` بدلالة البيانات فقط (بدون حاجة لتكرار).

#### 📐 المعادلة: الحل المغلق (Closed-form Solution)

$$
\hat{w}_1 = \frac{\sum y_i x_i - \dfrac{\sum y_i \sum x_i}{N}}{\sum x_i^2 - \dfrac{\left(\sum x_i\right)^2}{N}}
\quad , \quad
\hat{w}_0 = \frac{\sum y_i}{N} - \hat{w}_1 \frac{\sum x_i}{N}
$$

**الشرح:**
> - `Ŵ0` (تقدير المعامل الثابت / intercept): يساوي متوسط قيم `y` (average house price) مطروحاً منه `Ŵ1` مضروبة بمتوسط قيم `x` (average sq.ft.).
> - `Ŵ1` (تقدير الميل / slope): كسر يعتمد على مجموع حاصل ضرب `y` و `x`، ومجموع `x` و `y` منفصلَين، ومجموع مربعات `x`؛ كل ذلك مبني على بيانات التدريب فقط.
> - لاحظ أن `Ŵ1` لا بد أن يُحسب **أولاً** لأن `Ŵ0` يعتمد على قيمته.

#### مهم للامتحان ⚠️:
> ترتيب الحل ضروري: يجب حساب `Ŵ1` أولاً (من معادلة الميل)، ثم نعوّض به في معادلة `Ŵ0`. لو حاولت حساب `Ŵ0` أولاً بدون معرفة `Ŵ1` فلن تستطيع.

#### النص الأصلي يقول:
> "Approach 2: Gradient descent algorithm. For most ML problems, cannot solve gradient = 0. Even if solving gradient = 0 is feasible, gradient descent can be more efficient."

#### الشرح المبسّط:
في `Simple Linear Regression` (خط واحد بمعاملين فقط)، حل المعادلة الجبرية المباشرة (Closed-form) ممكن وسهل. لكن في أغلب مسائل تعلم الآلة الحقيقية (نماذج معقدة بمئات أو آلاف المعاملات)، **حل التدرج=صفر جبرياً يكون مستحيلاً عملياً أو مكلفاً حسابياً جداً**؛ لذلك `Gradient Descent` هي الطريقة الأعم والأكثر استخداماً في تعلم الآلة، حتى لو كان الحل المغلق موجوداً نظرياً.

```algorithm
1 | ابدأ بقيم ابتدائية لـ w0 و w1 | تحديد نقطة انطلاق | (w0^(0), w1^(0))
2 | احسب تدرج RSS عند القيم الحالية | تفاضل جزئي | ∇RSS(w0^(t), w1^(t))
3 | حدّث المعاملين معاً بعكس اتجاه التدرج | Gradient Descent rule | [w0,w1]^(t+1) = [w0,w1]^(t) - η∇RSS
4 | كرر حتى يصغر مقدار التدرج عن ε | فحص التقارب | ||∇RSS|| < ε
```

#### ⚖️ المقايضة: Closed-form Solution مقابل Gradient Descent

| | Closed-form (Set gradient=0) | Gradient Descent |
| --- | --- | --- |
| المزايا | حل دقيق ومباشر بخطوة واحدة (لا تكرار) | يعمل حتى لو استحال الحل الجبري، مرن مع أي دالة `convex` |
| العيوب | غير قابل للحل جبرياً في أغلب النماذج المعقدة (كثيرة المعاملات) | يحتاج ضبط `η` وعدد تكرارات، وقد يتقارب ببطء |
| متى تختاره | عندما يكون النموذج بسيطاً (مثل `Simple Linear Regression`) | عندما يكون النموذج كبيراً/معقداً (أغلب نماذج تعلم الآلة) |

### 9. الانحدار المتعدد (Multiple Regression)

#### النص الأصلي يقول:
> "Multiple regression, is linear regression with multiple features. Each feature is some function of either a single input, or multiple inputs. Two cases: 1. we have a single input and we're trying to use more complex functions of that single input to predict the output E.g. Polynomial regression. 2. we have multiple inputs."

#### الشرح المبسّط:
حتى الآن كان عندنا مُدخل (feature) واحد بسيط هو `x` (المساحة). لكن `Multiple Regression` تعني ببساطة: **انحدار خطي، لكن باستخدام عدة "features" بدل واحد فقط**. المهم أن كل `feature` يمكن أن يكون:
- **دالة على نفس المُدخل الواحد** (مثل `x`, `x²`, `x³`...) → هذه حالة `Polynomial Regression`.
- **أو مُدخلات مختلفة تماماً** (مثل عدد الغرف، عمر البيت، الموقع...) → هذه حالة `multiple inputs` الحقيقية.

**لماذا هذا التعميم مهم؟** لأن العلاقة الحقيقية بين المدخلات والمخرجات نادراً ما تكون خطاً مستقيماً بسيطاً؛ `Multiple Regression` تسمح لنا بنمذجة علاقات أعقد بكثير مع البقاء ضمن إطار "الانحدار الخطي" (لأن النموذج لا يزال خطياً بالنسبة للمعاملات `w`، حتى لو لم يكن خطياً بالنسبة للمدخل `x`).

#### 💡 التشبيه:
> تخيل أنك تحاول التنبؤ بسعر بيت، والخط المستقيم البسيط مثل استخدام "مسطرة واحدة" لقياس كل شيء. لكن الواقع أعقد (مساحة + عدد غرف + موقع...)، فبدل مسطرة واحدة، نستخدم عدة "أدوات قياس" (features) في نفس الوقت لنحصل على تنبؤ أدق.
> **وجه الشبه:** المسطرة الواحدة = `Simple Linear Regression` (feature واحد)، مجموعة أدوات القياس = `Multiple Regression` (عدة features).

### 10. الانحدار متعدد الحدود (Polynomial Regression)

#### النص الأصلي يقول:
> "Back to our case study, instead of using a simple line we can Fit our data with a quadratic function or Even higher order polynomial." ... "polynomial regression model: yi = w0 + w1xi + w2xi² + ... + wp xi^p + εi" (treat as different features)

#### الشرح المبسّط:
بدل أن يكون النموذج خطاً مستقيماً `y = w0 + w1*x`، يمكننا استخدام دالة من الدرجة الثانية (`Quadratic`: تحتوي `x²`) أو حتى درجة أعلى (`Higher Order Polynomial`: تحتوي `x², x³, ..., x^p`) لتناسب انحناءات في البيانات لا يستطيع الخط المستقيم تمثيلها.

**الفكرة الذكية هنا:** نتعامل مع `x`, `x²`, `x³`, ...، إلخ **كأنها features منفصلة تماماً** (وكأنها أعمدة مختلفة في جدول بيانات)، وبالتالي تبقى المعادلة **خطية بالنسبة للمعاملات** `w0, w1, ..., wp` حتى لو لم تكن خطية بالنسبة لـ `x` نفسه. هذا يسمح لنا باستخدام **نفس** أدوات الحل (`Closed-form` أو `Gradient Descent`) التي استخدمناها للخط البسيط.

#### 📐 المعادلة: نموذج Polynomial Regression

$$
y_i = w_0 + w_1 x_i + w_2 x_i^2 + \dots + w_p x_i^p + \varepsilon_i
$$

**الشرح:**
> - `εi` (epsilon): حد الخطأ العشوائي (noise term) الذي يمثل الفرق بين النموذج المثالي والقيمة الحقيقية بسبب عوامل غير مُقاسة.
> - كل حد `x^k` يُعامَل كـ feature مستقل: `feature 1 = 1 (constant)`, `feature 2 = x`, `feature 3 = x²`, ...، `feature p+1 = x^p`.
> - كل feature له معامله الخاص: `parameter 1 = w0`, `parameter 2 = w1`, ...، `parameter p+1 = wp`.

#### مهم للامتحان ⚠️:
> `Polynomial Regression` لا تزال تُعتبر **نموذجاً خطياً (Linear Model)** رغم وجود `x²` أو `x³` فيها! السبب: الخطية هنا تُقاس بالنسبة **للمعاملات w**، وليس بالنسبة للمُدخل `x`. هذا سوء فهم شائع جداً في الامتحانات.

#### 🤔 تفعيل الفهم (اسأل نفسك):
> **سؤال:** هل `y = w0 + w1*x + w2*x²` نموذج خطي أم غير خطي؟
> **لماذا هذا مهم؟** لأن الإجابة الصحيحة (خطي بالنسبة لـ w) تحدد أي أدوات حل (Closed-form/Gradient Descent) يمكن استخدامها مباشرة دون تعديل.

### 11. دوال أخرى للمُدخل الواحد (Other Functions of One Input): الاتجاه والموسمية (Trend & Seasonality)

#### النص الأصلي يقول:
> "Example: Two effects must be modeled. Trends over time: On average, house prices tend to increase with time. Seasonality: higher prices in the summer and lower prices in other months (repeated pattern over some period of time)."

#### الشرح المبسّط:
مثال عملي آخر على استخدام دوال أعقد من مُدخل واحد (الزمن `t` هنا بدل المساحة): أسعار البيوت عبر الزمن فيها **تأثيران منفصلان** يجب نمذجتهما معاً:
1. **Trend (اتجاه عام)**: مع مرور الوقت، الأسعار ترتفع بشكل عام تدريجياً (خط بطيء الميل).
2. **Seasonality (موسمية)**: نمط متكرر كل فترة زمنية ثابتة (مثلاً كل 12 شهراً)؛ أسعار الصيف أعلى من باقي الأشهر بسبب أن أغلب البيوت تُعرض للبيع في الصيف والبيوت الجيدة تُباع بسرعة، بينما القليل يُعرض في نوفمبر/ديسمبر وغالباً ما تكون معاملات خاصة أو مخزون متبقٍ.

**لماذا نحتاج نمذجة الاثنين معاً؟** لأن الاعتماد على `Trend` وحده يفوّت النمط المتكرر السنوي، والاعتماد على `Seasonality` وحده يفوّت الاتجاه العام طويل المدى؛ الجمع بينهما يعطي تمثيلاً أدق للواقع.

#### النص الأصلي يقول:
> "yi = w0 + w1ti + w2 sin(2πti/12 - Φ) + εi ... Equivalently, yi = w0 + w1ti + w2 sin(2πti/12) + w3 cos(2πti/12) + εi"

#### الشرح المبسّط:
لنمذجة `Trend` نستخدم حداً خطياً بسيطاً `w1*t`. أما لنمذجة `Seasonality` (نمط متكرر دوري)، نستخدم دالة جيبية (`sin`) بفترة زمنية `12` (تتكرر كل سنة إذا كانت `t` بالأشهر)، مع إزاحة طور غير معروفة `Φ` (Unknown phase/shift) تحدد "متى" تبدأ الذروة الموسمية بالضبط.

المشكلة: `Φ` داخل دالة `sin` يجعل النموذج **غير خطي** بالنسبة للمعاملات (لأنه لا يمكن فصل `Φ` كمعامل مستقل بسهولة). الحل: نستخدم **الهوية المثلثية (Trigonometric identity)**:
`sin(a - b) = sin(a)cos(b) - cos(a)sin(b)`

فنعيد كتابة `sin(2πt/12 - Φ)` كـ `sin(2πt/12)*cos(Φ) - cos(2πt/12)*sin(Φ)`. بما أن `cos(Φ)` و `sin(Φ)` مجرد **ثوابت** (لا تعتمد على `t`)، يمكننا تسميتها معاملات جديدة (`w2` و `w3`)، فيصبح النموذج خطياً بالكامل بالنسبة للمعاملات مرة أخرى!

#### 📐 المعادلة: نموذج Trend + Seasonality (بعد إعادة الصياغة الخطية)

$$
y_i = w_0 + w_1 t_i + w_2 \sin(2\pi t_i/12) + w_3 \cos(2\pi t_i/12) + \varepsilon_i
$$

**الشرح:**
> - `w0`: المعامل الثابت (intercept).
> - `w1*ti`: يمثل `Trend` (الاتجاه الخطي العام عبر الزمن).
> - `w2*sin(...) + w3*cos(...)`: يمثلان معاً `Seasonality` بفترة `12` (شهراً)، حيث تحمل `w2` و `w3` معلومة السعة (amplitude) وإزاحة الطور (phase shift) بشكل غير مباشر بعد فك الهوية المثلثية.
> - `feature 1 = 1` (ثابت)، `feature 2 = t`، `feature 3 = sin(2πt/12)`، `feature 4 = cos(2πt/12)` — أربعة features مستقلة يمكن حل معاملاتها بنفس أدوات الانحدار الخطي المعتادة.

#### 💡 التشبيه:
> تخيل تسجيل درجة حرارة الجو يومياً على مدار سنوات: هناك اتجاه عام بطيء (مثلاً بسبب التغير المناخي)، وفوقه نمط متكرر كل سنة (صيف حار، شتاء بارد). لفهم الحرارة في أي يوم، تحتاج الاثنين: خط الاتجاه البطيء + الموجة المتكررة السنوية.
> **وجه الشبه:** الاتجاه المناخي البطيء = `Trend`، التكرار الفصلي السنوي = `Seasonality`.

#### 📊 المخطط: مكونات نموذج Trend + Seasonality

#### ما هذا المخطط؟
> يوضح كيف يتحول نموذج `y = w0 + w1*t + w2*sin(2πt/12 - Φ) + ε` غير الخطي (بسبب Φ) إلى نموذج خطي بالكامل عبر الهوية المثلثية، ليصبح قابلاً للحل بنفس أدوات الانحدار الخطي.

```diagram
    level: 2
  - id: final
    label: "y = w0+w1t+w2sin(2πt/12)+w3cos(2πt/12)+ε"
    kind: event
    level: 3
  - id: solve
    label: حل بـ Closed-form أو Gradient Descent
    kind: event
    level: 4
edges:
  - from: original
    to: identity
  - from: identity
    to: rename
  - from: rename

  - from: final

```

| المصطلح | التعريف | مثال/ملاحظة |
| --- | --- | --- |

| `Convex Function` | دالة على شكل وعاء، أي مستقيم بين نقطتين عليها يقع فوقها أو مساوٍ لها | تضمن حلاً وحيداً للتصغير |
| `Concave Function` | دالة على شكل قبة، عكس Convex | تضمن حلاً وحيداً للتكبير |
| `Gradient` | متجه يحوي كل المشتقات الجزئية لدالة متعددة المتغيرات | تعميم المشتقة لأكثر من بُعد |

## خطوات وإجراءات المحاضرة

#### ⚙️ الخطوات / الخوارزمية: Hill Climbing (البحث عن max)

#### ما هدف هذه العملية؟
> إيجاد القيمة العظمى (max) لدالة أحادية البعد باستخدام إشارة المشتقة كدليل اتجاه.

```algorithm
1 | تهيئة | تحديد w ابتدائي | نبدأ من قيمة عشوائية w^(0)
2 | حساب المشتقة | تفاضل | نحسب dg/dw عند w^(t) الحالية
3 | تحديث القيمة | Hill Climbing rule | w^(t+1) = w^(t) + η * dg/dw
4 | فحص التقارب | معيار التوقف | نتوقف إذا |dg/dw| < ε
```

#### نقاط التنفيذ:
- التحديث بـ **إضافة** حاصل ضرب η والمشتقة (وليس طرحها).
- يستخدم فقط عندما نبحث عن **أعلى** نقطة.

---

#### ⚙️ الخطوات / الخوارزمية: Hill Descent (البحث عن min)

#### ما هدف هذه العملية؟
> إيجاد القيمة الصغرى (min) لدالة أحادية البعد.

```algorithm
1 | تهيئة | تحديد w ابتدائي | نبدأ من قيمة عشوائية w^(0)
2 | حساب المشتقة | تفاضل | نحسب dg/dw عند w^(t) الحالية
3 | تحديث القيمة | Hill Descent rule | w^(t+1) = w^(t) - η * dg/dw
4 | فحص التقارب | معيار التوقف | نتوقف إذا |dg/dw| < ε
```

#### نقاط التنفيذ:
- التحديث بـ **طرح** حاصل ضرب η والمشتقة.
- هي الأساس المباشر لخوارزمية `Gradient Descent`.

---

#### ⚙️ الخطوات / الخوارزمية: Gradient Descent لحل Simple Linear Regression

#### ما هدف هذه العملية؟
> إيجاد المعاملين `Ŵ0` و `Ŵ1` اللذين يُصغّران `RSS(w0,w1)` بطريقة تكرارية، مناسبة حتى عندما يستحيل الحل الجبري المباشر.

```algorithm
1 | تهيئة المعاملات | تحديد نقطة انطلاق | w0^(0), w1^(0) بقيم ابتدائية (غالباً أصفار أو عشوائية)
2 | حساب التوقعات | Forward pass | ŷi = w0^(t) + w1^(t) * xi لكل نقطة بيانات
3 | حساب التدرج | Backward / تفاضل جزئي | ∇RSS(w0,w1) بالصيغتين لـ w0 و w1
4 | تحديث المعاملين معاً | Gradient Descent rule | [w0,w1]^(t+1) = [w0,w1]^(t) - η∇RSS(w0^(t),w1^(t))
5 | فحص التقارب | معيار التوقف | توقف إذا ‖∇RSS(w0,w1)‖ < ε، وإلا كرر من الخطوة 2
```

#### نقاط التنفيذ:
- يجب حساب التدرج باستخدام **نفس** قيم `w0^(t), w1^(t)` القديمة قبل التحديث (لا نحدّث w0 ثم نستخدمه في حساب تدرج w1 بنفس التكرار).
- اختيار `η` غير مناسب (كبير جداً) قد يمنع التقارب كلياً (تذبذب أو تباعد).

---

#### ⚙️ الخطوات / الخوارزمية: الحل المغلق (Closed-form) لـ Simple Linear Regression

#### ما هدف هذه العملية؟
> إيجاد `Ŵ0` و `Ŵ1` مباشرة بمعادلة جبرية واحدة، دون الحاجة لأي تكرار، لأن `RSS` هنا دالة بسيطة بمتغيرين فقط.

```algorithm
1 | حساب المجاميع الأساسية | إحصائيات وصفية | Σyi, Σxi, Σ(yi*xi), Σ(xi²), N
2 | حساب Ŵ1 (الميل) | تعويض مباشر بالصيغة | Ŵ1 = [Σyixi - (Σyi*Σxi)/N] / [Σxi² - (Σxi)²/N]
3 | حساب Ŵ0 (الثابت) | تعويض مباشر بالصيغة | Ŵ0 = (Σyi/N) - Ŵ1*(Σxi/N)
```

#### نقاط التنفيذ:
- لا يوجد `step size` ولا `convergence criteria` هنا لأن الحل مباشر وليس تكرارياً.
- يصلح فقط عندما نستطيع فعلياً حل نظام المعادلات جبرياً (نماذج بسيطة نسبياً).

## أنماط الأكواد الشائعة

#### 💻 الكود: تنفيذ Gradient Descent لـ Simple Linear Regression بلغة Python

#### ما هذا الكود؟
> يطبّق خوارزمية `Gradient Descent` المشروحة أعلاه لإيجاد `w0` و `w1` اللذين يُصغّران `RSS` على بيانات (مساحة البيت، السعر).

```python
import numpy as np  # numerical operations on arrays

def gradient_descent(x, y, eta=0.0000001, epsilon=1e-6, max_iter=10000):
    # x: array of square footage values
    # y: array of house prices
    N = len(x)  # number of training points
    w0, w1 = 0.0, 0.0  # initial parameter values
    for t in range(max_iter):  # iterate until convergence or max_iter
        y_hat = w0 + w1 * x  # predicted values (vectorized)
        residual = y - y_hat  # residual for each point
        grad_w0 = -2 * np.sum(residual)  # partial derivative w.r.t. w0
        grad_w1 = -2 * np.sum(residual * x)  # partial derivative w.r.t. w1
        w0 = w0 - eta * grad_w0  # update w0 opposite to gradient
        w1 = w1 - eta * grad_w1  # update w1 opposite to gradient
        grad_norm = np.sqrt(grad_w0**2 + grad_w1**2)  # magnitude of gradient
        if grad_norm < epsilon:  # convergence check
            break  # stop iterating, we are close enough to optimum
    return w0, w1  # return estimated parameters
```

#### شرح كل سطر:
1. `import numpy as np` → استيراد المكتبة — لعمليات المصفوفات السريعة (vectorized).
2. `def gradient_descent(...)` → تعريف الدالة — تأخذ البيانات ومعاملات التحكم (η, ε, عدد التكرارات).
3. `N = len(x)` → عدد نقاط البيانات — يستخدم ضمنياً عبر `np.sum` على المصفوفات.
4. `w0, w1 = 0.0, 0.0` → تهيئة المعاملات — نقطة انطلاق الخوارزمية.
5. `for t in range(max_iter):` → حلقة التكرار — تمثل "while not converged" مع حد أقصى للأمان.
6. `y_hat = w0 + w1 * x` → حساب التوقعات — تطبيق معادلة الخط على كل نقاط `x` دفعة واحدة.
7. `residual = y - y_hat` → حساب البواقي — الفرق بين الحقيقي والمتوقع لكل نقطة.
8. `grad_w0 = -2 * np.sum(residual)` → تطبيق صيغة تدرج w0 مباشرة.
9. `grad_w1 = -2 * np.sum(residual * x)` → تطبيق صيغة تدرج w1 مباشرة (بضرب البواقي بـ x).
10. `w0 = w0 - eta * grad_w0` → قاعدة تحديث Gradient Descent لـ w0.
11. `w1 = w1 - eta * grad_w1` → قاعدة تحديث Gradient Descent لـ w1.
12. `grad_norm = np.sqrt(...)` → حساب مقدار (magnitude) متجه التدرج.
13. `if grad_norm < epsilon: break` → معيار التقارب — التوقف عند الاقتراب الكافي من الحل.
14. `return w0, w1` → إرجاع المعاملات النهائية المقدَّرة.

**المكتبات المطلوبة (Imports):**
> `import numpy as np`

**الناتج المتوقع:**
> قيمتان عدديتان `w0` و `w1` تقتربان تدريجياً من الحل المغلق الدقيق، بشرط اختيار `eta` صغير بما يكفي لتفادي التذبذب (وقد تحتاج قيمة صغيرة جداً لأن مقياس x بالـ sq.ft. كبير عادة).

🛠️ **استكشاف الأخطاء:**
| الخطأ | السبب | الحل |
| --- | --- | --- |
| القيم تتباعد (diverge) لانهائياً | `eta` كبير جداً بالنسبة لحجم البيانات | تصغير `eta` أو استخدام `decreasing step size` |
| بطء شديد في التقارب | `eta` صغير جداً أو `epsilon` صغير جداً جداً | زيادة `eta` قليلاً أو تخفيف `epsilon` |
| النتائج NaN | تجاوز حسابي (overflow) بسبب eta كبير مع بيانات كبيرة القيم | توحيد قياس البيانات (normalization) قبل التدريب |

#### 💻 الكود: بناء features لـ Polynomial Regression بلغة Python

#### ما هذا الكود؟
> يجهّز مصفوفة features (`x, x², ..., x^p`) من مُدخل واحد لاستخدامها في `sklearn` كنموذج انحدار خطي عادي.

```python
import numpy as np  # for array operations
from sklearn.linear_model import LinearRegression  # linear regression model

def build_polynomial_features(x, degree):
    # x: 1D array of a single input feature
    # degree: highest power p to include
    N = len(x)  # number of data points
    X = np.ones((N, degree + 1))  # start with constant feature (all ones)
    for p in range(1, degree + 1):  # loop over powers 1..degree
        X[:, p] = x ** p  # fill column p with x raised to power p
    return X  # matrix of shape (N, degree+1)

x = np.array([1000, 1500, 2000, 2500])  # example sq.ft. values
y = np.array([200000, 250000, 320000, 340000])  # example prices
X_poly = build_polynomial_features(x, degree=2)  # build quadratic features
model = LinearRegression(fit_intercept=False)  # intercept already included as feature 1
model.fit(X_poly, y)  # fit w0, w1, w2 via least squares
print(model.coef_)  # print estimated [w0, w1, w2]
```

#### شرح كل سطر:
1. `import numpy as np` → للتعامل مع المصفوفات.
2. `from sklearn.linear_model import LinearRegression` → استيراد نموذج الانحدار الخطي الجاهز.
3. `def build_polynomial_features(...)` → دالة تبني مصفوفة features من مُدخل واحد.
4. `X = np.ones((N, degree+1))` → إنشاء مصفوفة تبدأ كلها بـ 1 (تمثل feature الثابت w0).
5. `for p in range(1, degree+1):` → حلقة على كل قوة من 1 إلى degree.
6. `X[:, p] = x ** p` → ملء العمود رقم p بقيمة `x` مرفوعة للقوة `p`.
7. `return X` → إرجاع مصفوفة الـ features الكاملة.
8. `x = np.array([...])` → بيانات مساحة تجريبية.
9. `y = np.array([...])` → بيانات سعر تجريبية مقابلة.
10. `X_poly = build_polynomial_features(x, degree=2)` → بناء features تربيعية (Quadratic).
11. `model = LinearRegression(fit_intercept=False)` → لأن العمود الأول من X_poly يمثل الثابت أصلاً.
12. `model.fit(X_poly, y)` → تدريب النموذج بطريقة `Least Squares` (تعادل حل RSS).
13. `print(model.coef_)` → طباعة المعاملات المقدَّرة [w0, w1, w2].

**المكتبات المطلوبة (Imports):**
> `numpy`, `sklearn.linear_model.LinearRegression`

**الناتج المتوقع:**
> مصفوفة معاملات `[w0, w1, w2]` تمثل أفضل دالة تربيعية تُقارب البيانات المُعطاة حسب معيار `RSS`.

## أنماط التعامل (Patterns)

- **نمط "تحويل غير الخطي إلى خطي"**: عند مواجهة دالة تحتوي معاملاً داخل دالة غير خطية (مثل `Φ` داخل `sin`)، ابحث عن هوية رياضية (مثلية أو غيرها) تفصل الثابت المجهول كمعامل ضرب خارجي.
- **نمط "المعالجة كـ features منفصلة"**: أي تعقيد في المُدخل (`x²`, `sin(t)`, `cos(t)`) يمكن التعامل معه كعمود بيانات مستقل، طالما بقي النموذج خطياً بالنسبة للمعاملات `w`.
- **نمط "التكرار حتى التقارب"**: أي خوارزمية `Hill Climbing/Descent/Gradient Descent` تتبع نفس الهيكل: تهيئة → حساب المشتقة/التدرج → تحديث → فحص تقارب → تكرار.

## الأفكار الشاملة (Big Picture)

بناء أي نموذج انحدار يمر بنفس الدورة: **(1)** اختيار شكل النموذج ونوع الـ features، **(2)** تعريف مقياس جودة (`RSS`)، **(3)** صياغة المشكلة كتحسين رياضي (تصغير `RSS`)، **(4)** حل هذا التحسين إما بمعادلة مباشرة (`Closed-form`) أو بخوارزمية تكرارية (`Gradient Descent`). هذه الدورة الأربعية هي نفسها التي ستتكرر لاحقاً في `Classification` و`Clustering` مع تغيّر شكل دالة الخطأ فقط.

## الجزء الثالث: أسئلة اختيار من متعدد (MCQ)

### السؤال 1
لماذا نستخدم `RSS` بدلاً من مجموع الأخطاء المطلقة (Sum of Absolute Errors) كمقياس جودة؟

أ. لأن التربيع أسهل حسابياً من القيمة المطلقة
ب. لأن التربيع يجعل الدالة قابلة للاشتقاق بسلاسة ويمنع إلغاء الأخطاء الموجبة والسالبة لبعضها

### سؤال تصحيح 1 (النوع: `wrong_formula`)


**اكتشف الخطأ:** الكود يحسب تدرج `w1` بنفس صيغة تدرج `w0` تماماً، متجاهلاً الضرب بـ `x` الذي تتطلبه صيغة `∇RSS` (لأن `w1` تُضرب بـ `xi` في النموذج الأصلي).

**شرح الحل:**

### سؤال تصحيح 2 (النوع: `logic`)

**الكود (يحتوي خطأ):**
```python
def gradient_descent_step(w0, w1, grad_w0, grad_w1, eta):
    # WRONG: moving in the same direction as gradient (like hill climbing)
    w0_new = w0 + eta * grad_w0
    w1_new = w1 + eta * grad_w1
    return w0_new, w1_new
```

**اكتشف الخطأ:** الكود يستخدم علامة **جمع** (+) عند التحديث، وهذا يطبّق قاعدة `Hill Climbing` (البحث عن max)، بينما هدفنا هنا هو **تصغير** RSS (Hill/Gradient **Descent**)، لذا يجب استخدام الطرح.

**التصحيح:**
```python
def gradient_descent_step(w0, w1, grad_w0, grad_w1, eta):
    # CORRECT: moving opposite to the gradient (gradient descent)
    w0_new = w0 - eta * grad_w0
    w1_new = w1 - eta * grad_w1
    return w0_new, w1_new
```

**شرح الحل:**
1. `RSS` هي دالة نريد **تصغيرها**، وليس تكبيرها.
2. قاعدة `Gradient Descent` تتطلب التحرك **بعكس** اتجاه التدرج (طرح وليس جمع).
3. استخدام الجمع بالخطأ سيجعل الخوارزمية تتحرك نحو **زيادة** RSS بدلاً من تصغيرها، أي تبتعد عن الحل الأمثل.

```python
    # WRONG: forgets to divide by N, and forgets to subtract w1 term
    w0_hat = sum_y - w1 * sum_x
    return w0_hat
```

**اكتشف الخطأ:** الدالة تُرجع صيغة خاطئة تماماً؛ الصيغة الصحيحة لـ `Ŵ0` هي `(Σy/N) - Ŵ1*(Σx/N)`، بينما الكود يستخدم `Σy - Ŵ1*Σx` بدون القسمة على `N` إطلاقاً في أي من الحدين.

**التصحيح:**
```python
def estimate_w0(y, x, w1, N):
    sum_y = sum(y)
    sum_x = sum(x)
    # CORRECT: average of y minus w1 times average of x
    w0_hat = (sum_y / N) - w1 * (sum_x / N)
    return w0_hat
```

**شرح الحل:**
1. صيغة `Ŵ0` تعتمد على **متوسط** `y` ومتوسط `x` (average house price وaverage sq.ft.)، وليس المجموع الخام.
2. نسيان القسمة على `N` يجعل القيمة المُرجَعة أكبر بكثير من الحل الصحيح وغير منطقية إحصائياً.
3. يجب دائماً التأكد أن `Ŵ1` قد حُسب مسبقاً وتم تمريره صحيحاً قبل استدعاء هذه الدالة.

### سؤال تصحيح 5 (النوع: `dead_code`)

**الكود (يحتوي خطأ):**
```python

    w0, w1 = 0.0, 0.0
    converged = False
    for t in range(max_iter):
        y_hat = w0 + w1 * x
        residual = y - y_hat

        grad_w1 = -2 * sum(residual * x)
        w0 = w0 - eta * grad_w0
        w1 = w1 - eta * grad_w1
        # DEAD CODE: this check never actually breaks the loop
        if converged:
            break
        grad_norm = (grad_w0**2 + grad_w1**2) ** 0.5
        # converged is never updated based on grad_norm!

```

**اكتشف الخطأ:** المتغير `converged` يُعرَّف بقيمة `False` ولا يُحدَّث أبداً داخل الحلقة بناءً على `grad_norm`؛ لذلك شرط `if converged: break` هو كود ميت (dead code) لن يُنفَّذ أبداً، والحلقة ستستمر حتى `max_iter` دائماً بدون أي فحص حقيقي للتقارب.

**التصحيح:**
```python
def train_gradient_descent(x, y, eta, epsilon, max_iter):
    w0, w1 = 0.0, 0.0
    for t in range(max_iter):
        y_hat = w0 + w1 * x
        residual = y - y_hat
        grad_w0 = -2 * sum(residual)
        grad_w1 = -2 * sum(residual * x)
        w0 = w0 - eta * grad_w0
        w1 = w1 - eta * grad_w1
        grad_norm = (grad_w0**2 + grad_w1**2) ** 0.5
        # CORRECT: actually check the convergence criterion each iteration
        if grad_norm < epsilon:
            break
    return w0, w1
```

**شرح الحل:**
1. معيار التقارب الحقيقي يجب أن يُفحص **بعد** حساب `grad_norm` مباشرة، وليس عبر متغير لا يتغيّر أبداً.
2. الكود الأصلي يهدر فرصة التوقف المبكر (early stopping)، مما يجعل التدريب يستمر لعدد تكرارات غير ضروري.
3. القاعدة الصحيحة: `if grad_norm < epsilon: break` يجب أن تُستخدم مباشرة بدل الاعتماد على متغير وهمي ثابت.

## الجزء الخامس: أسئلة نظرية متوقعة بالامتحان

### السؤال 1: ما هو RSS ولماذا نستخدمه؟
**نموذج الإجابة:** 1. التعريف: مجموع مربعات الفروقات بين القيم الحقيقية والمتوقعة. 2. المكونات: residual لكل نقطة، تربيعه، ثم الجمع على كل نقاط التدريب N. 3. مثال رقمي: بيانات (x,y) بسيطة كما في تمرين حساب RSS أعلاه. 4. متى نستخدمه: كمقياس جودة لاختيار أفضل خط/نموذج انحدار قبل البحث عن الحل الأمثل.

### السؤال 2: اشرح الفرق بين Approach 1 (Set gradient=0) وApproach 2 (Gradient Descent).
**نموذج الإجابة:** 1. التعريف: الأول حل جبري مباشر، الثاني خوارزمية تكرارية. 2. الشروط: الأول يتطلب إمكانية حل نظام المعادلات جبرياً، الثاني يعمل حتى لو استحال ذلك. 3. مثال: Simple Linear Regression يمكن حله بالطريقتين، بينما نماذج معقدة تحتاج الطريقة الثانية فقط. 4. متى نستخدم كل واحد: الأول للنماذج البسيطة، الثاني للنماذج الكبيرة/المعقدة.

### السؤال 3: ما شروط ضمان حل وحيد عند مساواة المشتقة بصفر؟
**نموذج الإجابة:** 1. التعريف: يجب أن تكون الدالة إما convex (للتصغير) أو concave (للتكبير). 2. المكونات: خاصية أن الخط الواصل بين أي نقطتين على المنحنى يقع فوقه (concave) أو تحته (convex). 3. مثال: RSS كدالة في w0,w1 هي convex، لذا حلها وحيد. 4. متى تفشل هذه الضمانة: عندما تكون الدالة "Neither" فتظهر حلول محلية متعددة.

### السؤال 4: عرّف الـ Gradient ووضّح علاقته بالمشتقة العادية.
**نموذج الإجابة:** 1. التعريف: متجه يحوي كل المشتقات الجزئية لدالة متعددة المتغيرات. 2. المكونات: كل عنصر في المتجه هو مشتقة جزئية بالنسبة لمتغير واحد مع تثبيت الباقي. 3. مثال: ∇RSS(w0,w1) متجه بعنصرين. 4. متى نستخدمه: عندما يكون لدينا أكثر من معامل واحد لتحسينه في نفس الوقت.

### السؤال 5: لماذا يُعتبر Polynomial Regression نموذجاً خطياً رغم احتوائه على x²، x³؟
**نموذج الإجابة:** 1. التعريف: الخطية تُقاس بالنسبة للمعاملات w وليس المُدخل x. 2. المكونات: كل قوة من x (x, x², ...) تُعامل كـ feature منفصل يُضرب بمعامل واحد فقط. 3. مثال: y=w0+w1x+w2x² خطي في w0,w1,w2. 4. متى يهم هذا: عند اختيار طريقة الحل (Closed-form/Gradient Descent) التي تتطلب خطية بالمعاملات.

### السؤال 6: اشرح فكرة step size وأنواعه.
**نموذج الإجابة:** 1. التعريف: η يتحكم بمقدار التغيير في w بكل تكرار. 2. المكونات: Fixed step size (ثابت طوال التنفيذ) وDecreasing step size (يتناقص مع الزمن). 3. مثال: η_t=α/t أو η_t=α/√t. 4. متى نختار كل نوع: Fixed للبساطة، Decreasing لدقة أعلى قرب الحل مع الحذر من التصغير السريع جداً.

### السؤال 7: كيف تتحول معادلة الموسمية من غير خطية إلى خطية؟
**نموذج الإجابة:** 1. التعريف: استخدام هوية sin(a-b) لفصل الطور المجهول Φ. 2. المكونات: sin(2πt/12)cos(Φ) - cos(2πt/12)sin(Φ)، حيث cos(Φ) وsin(Φ) تصبحان معاملات ثابتة جديدة w2,w3. 3. مثال: y=w0+w1t+w2sin(2πt/12)+w3cos(2πt/12). 4. متى يفيد هذا: عند الحاجة لاستخدام أدوات الانحدار الخطي القياسية بدل حل غير خطي معقد.

### السؤال 8: ما معنى "التقارب" (Convergence) في خوارزميات التحسين التكرارية؟
**نموذج الإجابة:** 1. التعريف: الحالة التي تتوقف عندها الخوارزمية لأنها اقتربت كفاية من الحل الأمثل. 2. المكونات: عتبة ε تُقارَن بها قيمة المشتقة أو مقدار التدرج. 3. مثال: |dg/dw|<ε في بُعد واحد، ‖∇g(w)‖<ε في أبعاد متعددة. 4. متى نستخدمه: كشرط توقف في أي خوارزمية Hill Climbing/Descent/Gradient Descent.

### السؤال 9: قارن بين Trend وSeasonality من ناحية النمذجة الرياضية.
**نموذج الإجابة:** 1. التعريف: Trend اتجاه عام طويل المدى، Seasonality نمط متكرر دوري. 2. المكونات: Trend يُنمذج بحد خطي w1*t، Seasonality بحدود sin/cos بفترة معينة. 3. مثال: أسعار البيوت ترتفع عموماً (Trend) لكن ترتفع أكثر صيفاً (Seasonality). 4. متى نحتاج الجمع بينهما: عندما تظهر البيانات كلا النمطين معاً كما في مثال أسعار البيوت الشهرية.

### السؤال 10: اشرح لماذا نربّع الأخطاء في RSS بدل استخدام القيمة المطلقة.
**نموذج الإجابة:** 1. التعريف: RSS = مجموع مربعات residuals. 2. المكونات: التربيع يمنع إلغاء الأخطاء الموجبة والسالبة، ويعطي دالة قابلة للاشتقاق بسلاسة في كل نقطة (بخلاف |.| التي لها نقطة غير قابلة للاشتقاق عند الصفر). 3. مثال: خطأ +5 وخطأ -5 لا يلغيان بعضهما بعد التربيع (25+25=50). 4. متى يهم هذا: عند استخدام أدوات التفاضل (Gradient Descent/Closed-form) لإيجاد الحل الأمثل.

### السؤال 11: ما الفرق بين Simple Linear Regression وMultiple Regression؟
**نموذج الإجابة:** 1. التعريف: الأول feature واحد بسيط (x)، الثاني عدة features (دوال معقدة لمدخل واحد أو مدخلات متعددة). 2. المكونات: كلاهما لا يزال "خطياً بالمعاملات w". 3. مثال: y=w0+w1x (بسيط) مقابل y=w0+w1x+w2x² (متعدد/Polynomial). 4. متى نستخدم كل واحد: البسيط عند علاقة خطية واضحة، المتعدد عند الحاجة لالتقاط انحناءات أو تأثيرات متعددة.

## الجزء الرابع: بطاقات سؤال وجواب (Q&A Cards)

**Q1:** ما هو RSS؟
A: مجموع مربعات الفروقات بين القيم الحقيقية والمتوقعة على كل بيانات التدريب.

**Q2:** ما الفرق بين Hill Climbing وHill Descent؟
A: Climbing يبحث عن max ويتحرك مع اتجاه المشتقة، Descent يبحث عن min ويتحرك بعكسها.

**Q3:** متى يكون حل derivative=0 وحيداً ومضموناً؟
A: عندما تكون الدالة convex (للتصغير) أو concave (للتكبير).

**Q4:** ما هو الـ Gradient؟
A: متجه يحوي كل المشتقات الجزئية لدالة متعددة المتغيرات.

**Q5:** ما معنى Partial Derivative؟
A: مشتقة الدالة بالنسبة لمتغير واحد فقط، مع اعتبار باقي المتغيرات ثوابت.

**Q6:** ما هو step size (η)؟
A: رقم يتحكم بمقدار تغيّر w في كل تكرار من خوارزمية التحسين.

**Q7:** ما الفرق بين Fixed وDecreasing step size؟
A: الثابت لا يتغير طوال التنفيذ، بينما المتناقص يصغر تدريجياً مع زيادة عدد التكرارات.

**Q8:** ما معيار التقارب في بُعد واحد؟
A: |dg/dw| < ε (القيمة المطلقة للمشتقة أصغر من عتبة صغيرة ε).

**Q9:** ما معيار التقارب في أبعاد متعددة؟
A: ‖∇g(w)‖ < ε (مقدار متجه التدرج أصغر من ε).

**Q10:** ما الفرق بين Closed-form Solution وGradient Descent؟
A: الأول حل جبري مباشر بخطوة واحدة، والثاني خوارزمية تكرارية تصلح لكل الحالات تقريباً.

**Q11:** ما هو Multiple Regression؟
A: انحدار خطي يستخدم أكثر من feature واحد، إما دوال معقدة لمُدخل واحد أو مُدخلات متعددة.

**Q12:** ما هو Polynomial Regression؟
A: نوع من Multiple Regression يستخدم قوى مختلفة لنفس المُدخل (x, x², ..., x^p) كـ features منفصلة.

**Q13:** هل Polynomial Regression نموذج خطي أم غير خطي؟
A: خطي بالنسبة للمعاملات w، رغم احتوائه على x², x³.

**Q14:** ما الفرق بين Trend وSeasonality؟
A: Trend اتجاه عام طويل المدى، وSeasonality نمط متكرر دورياً (مثل كل 12 شهراً).

**Q15:** لماذا نستخدم الهوية المثلثية sin(a-b) في نمذجة Seasonality؟
A: لتحويل النموذج من غير خطي (بسبب الطور Φ داخل sin) إلى نموذج خطي بالكامل بالنسبة للمعاملات.

**Q16:** ما ترتيب حساب Ŵ0 وŴ1 في الحل المغلق؟
A: يجب حساب Ŵ1 أولاً، ثم استخدامه لحساب Ŵ0.

## الجزء الخامس: كتابة الكود الكامل (مرجع شامل)

> الكود الكامل مجمّع من جميع أجزاء المحاضرة — مرجع للطالب، لا شرح جديد.

```python
# === Data Preprocessing (تحضير بسيط للبيانات) ===
import numpy as np

def load_example_data():
    # example square footage and price data
    x = np.array([1000, 1500, 2000, 2500, 3000], dtype=float)
    y = np.array([200000, 250000, 320000, 340000, 400000], dtype=float)
    return x, y

# === Regression: RSS computation ===
def compute_rss(w0, w1, x, y):
    y_hat = w0 + w1 * x  # predicted values
    residuals = y - y_hat  # errors
    return np.sum(residuals ** 2)  # sum of squared residuals

# === Regression: Gradient computation ===
def compute_gradient(w0, w1, x, y):
    y_hat = w0 + w1 * x
    residuals = y - y_hat
    grad_w0 = -2 * np.sum(residuals)
    grad_w1 = -2 * np.sum(residuals * x)
    return grad_w0, grad_w1

# === Regression: Gradient Descent training loop ===
def gradient_descent(x, y, eta=1e-8, epsilon=1e-3, max_iter=100000):
    w0, w1 = 0.0, 0.0
    for t in range(max_iter):
        grad_w0, grad_w1 = compute_gradient(w0, w1, x, y)
        w0 = w0 - eta * grad_w0
        w1 = w1 - eta * grad_w1
        grad_norm = np.sqrt(grad_w0 ** 2 + grad_w1 ** 2)
        if grad_norm < epsilon:
            break
    return w0, w1

# === Regression: Closed-form solution ===
def closed_form_solution(x, y):
    N = len(x)
    sum_x, sum_y = np.sum(x), np.sum(y)
    sum_xy, sum_x2 = np.sum(x * y), np.sum(x ** 2)
    w1_hat = (sum_xy - (sum_x * sum_y) / N) / (sum_x2 - (sum_x ** 2) / N)
    w0_hat = (sum_y / N) - w1_hat * (sum_x / N)
    return w0_hat, w1_hat

# === Polynomial Regression: feature construction ===
def build_polynomial_features(x, degree):
    N = len(x)
    X = np.ones((N, degree + 1))
    for p in range(1, degree + 1):
        X[:, p] = x ** p
    return X

# === Seasonality: feature construction (trend + seasonal) ===
def build_seasonal_features(t):
    N = len(t)
    X = np.ones((N, 4))
    X[:, 1] = t  # trend component
    X[:, 2] = np.sin(2 * np.pi * t / 12)  # seasonal sine component
    X[:, 3] = np.cos(2 * np.pi * t / 12)  # seasonal cosine component
    return X

# === Example usage ===
if __name__ == "__main__":
    x, y = load_example_data()
    w0_gd, w1_gd = gradient_descent(x, y)
    w0_cf, w1_cf = closed_form_solution(x, y)
    print("Gradient Descent:", w0_gd, w1_gd)
    print("Closed-form:", w0_cf, w1_cf)
```
